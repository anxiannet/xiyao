# 《夕妖：占山为王》专属卡牌边框设计

## 文件定位

本文件定义《占山为王》妖怪卡专属卡牌边框的视觉方案，用于后续提取 UI、制作透明 PNG 通用边框素材、Figma/Photoshop 合成与网页展示。

本文件只负责视觉设计，不定义：

- 世界观设定
- 角色事实
- 游戏规则
- 卡牌数量
- 技能效果
- 数值强度

规则、卡牌数据、技能文案仍以以下文件为唯一来源：

```text
游戏/卡牌游戏/占山为王/V1.2-基础卡表.md
游戏/卡牌游戏/占山为王/V1.2-卡面技能文案.md
```

UI 资产拆分与透明 PNG 要求以以下文件为统一来源：

```text
视觉设计/占山为王/UI资产规范.md
```

卡面结构与印刷要求以以下文件为统一来源：

```text
视觉设计/占山为王/卡牌框架规范.md
```

本文件是上述文件下的专项视觉方案，不另建平行规则体系。

---

# 设计目标

占山为王妖怪卡边框需要满足：

- 一眼区分 1 星、2 星、3 星
- 3 星有首领感，但不能提前消耗 4 星、5 星视觉空间
- 基础牌最高只到 3 星，视觉强度必须克制
- UI 与插画分离，便于批量合成
- 透明 PNG 可长期复用
- 手机缩略图下仍能识别星级、名字与是否有技能
- 适合实体印刷与网页展示

---

# 当前基础牌星级原则

当前基础牌只做到 3 星。

## 1 星小妖

视觉定位：普通守军、基础单位。

卡面展示：

- 1 颗星
- 名字
- 插画

不展示：

- 技能栏
- 技能图标
- 技能文案
- “小妖”文字标签

边框语言：

- 青铜 / 蓝铜方向
- 简洁角饰
- 无明显流光
- 无大型兽首、火焰、雷电、传奇特效

---

## 2 星精英

视觉定位：精英守军、骨干单位。

卡面展示：

- 2 颗星
- 名字
- 插画

不展示：

- 技能栏
- 技能图标
- 技能文案
- “精英”文字标签

边框语言：

- 银铁 / 紫银方向
- 比 1 星多少量妖纹
- 允许轻微金属高光
- 不使用 3 星红金主框
- 不使用首领级技能栏

---

## 3 星首领

视觉定位：基础牌当前最高层级。

卡面展示：

- 3 颗星
- 竖排名字
- 插画
- 底部技能文案

不展示：

- 技能图标
- “技能”二字
- “首领”二字
- “精英”二字
- “妖王”二字
- 技能类型标签
- 职业 / 属性圆形徽章

边框语言：

- 红金 / 暗金方向
- 四周连续金属边框
- 圆角竖排名字牌
- 底部短技能栏
- 少量流光即可
- 不使用龙头、巨大兽首、满屏火焰、雷电、爆闪特效

3 星必须靠边框复杂度、颜色、三颗星与技能栏来传达首领层级，不靠文字说明“首领”。

# 3 星卡牌 UI 标准版式

## 卡面比例

建议比例：

```text
插画区：75% 左右
技能区：10% 至 15%
边框与安全区：其余空间
```

技能区只容纳 1 行短文案，不应过高。

---

## 星级区

位置：左上角。

要求：

- 3 颗金属浮雕星
- 星星之间留出清晰间距
- 不放在顶部中央
- 不放在名字下方
- 不与名字牌重叠
- 不使用额外文字说明星级

---

## 竖排名字牌

位置：左侧内缩，压在插画层之上，但不得贴近裁切线。

形态：

- 竖排
- 圆角长牌
- 红底金边
- 金色浮雕字
- 上下可有小宝石或金属节点
- 牌面必须容纳 `金鼻白毛老鼠精` 七个中文字

要求：

- 名字完整可读
- 不裁切
- 不被花纹遮挡
- 不缩成不可读小字
- 不写额外身份标签
- 名字牌本体可作为 UI 资产，文字由合成脚本后置写入

说明：

当前 3 星模板采用竖排名字方案。若旧版文档中存在“长名优先横排”描述，以本专项方案的 3 星模板为当前设计方向，后续同步修订卡牌框架规范。

---

## 技能区

位置：底部。

形态：

- 卷轴 / 石碑 / 妖纹铭牌
- 米黄色或浅羊皮纸底
- 金属红金包边
- 高度克制，只服务一行文案

要求：

- 只出现技能效果文本
- 不出现“技能”二字
- 不出现技能图标
- 不出现技能名
- 不出现主动 / 被动 / 登场等类型标签
- 不使用普通矩形备注栏
- 不压缩插画主体

示例显示：

```text
偷1只1星守军来守地盘
```

最终文案必须来自：

```text
游戏/卡牌游戏/占山为王/V1.2-卡面技能文案.md
```

# 3 星边框视觉强度控制

3 星是当前基础牌最高层级，但不是未来最高稀有度。

允许：

- 红金主框
- 暗金金属厚度
- 角饰
- 圆角竖排名字牌
- 底部技能铭牌
- 少量妖纹
- 少量流光
- 小宝石节点

禁止：

- 巨大龙头
- 巨大兽首
- 满屏火焰
- 满屏雷电
- 传奇级爆闪
- 过度镭射
- 全边框高强度特效
- 过大的阵营旗帜压住主体
- 右上角无规则意义的职业 / 属性圆形徽章

复杂度参考：

```text
1星：30%
2星：50%
3星：70%
4星：85%
5星：100%
```

当前基础 3 星控制在 70% 左右，为未来 4 星、5 星、扩展卡、收藏卡保留升级空间。

---

# 未来扩展空间

## 4 星

保留给未来妖王级或扩展卡。

可增加：

- 妖王兽首
- 更强边框厚度
- 阵营图腾
- 明显妖气
- 更强宝石节点

## 5 星

保留给传说级或特殊扩展。

可增加：

- 白金 / 暗金复合框
- 全边框光效
- 专属卡背
- 专属背景
- 更强收藏卡工艺感

## 收藏卡

收藏卡不等于更高星级。

收藏卡可在同星级强度不变的情况下增加：

- 镭射
- 烫金
- 签名
- 异画
- 特殊卡面材质

# UI 资产拆分

3 星妖怪卡建议拆分为以下透明 PNG 资产：

```text
视觉设计/占山为王/UI资产/monster/
├── ui_monster_frame_3star.png
├── ui_monster_name_plate_vertical_3star.png
├── ui_monster_skill_plate_3star.png
└── ui_star_3.png
```

资产要求：

- PNG
- 透明背景
- 不含角色
- 不含场景插画
- 不含名字文字
- 不含技能文字
- 不含“技能”二字
- 不含技能图标
- 不含“首领”二字
- 不含水印

最终卡牌合成顺序：

```text
插画底图
↓
3星边框
↓
3星星级
↓
竖排名字牌
↓
底部技能铭牌
↓
名字文字
↓
技能文字
```

# 通用边框素材生成提示词方向

用于生成 3 星通用 UI 边框时，应强调：

```text
竖版 63:88 collectible game card UI frame, transparent background, red and dark gold metal frame, rounded corners, ornate but restrained eastern fantasy pattern, vertical rounded name plate on left side, bottom parchment skill plate, three-star area reserved at top-left, no character, no illustration, no text, no skill icon, no labels, no dragon head, no huge beast head, no excessive flames, no lightning, clean reusable UI asset, premium but not legendary, suitable for 3-star base card
```

```text
竖版63:88卡牌UI通用边框，透明背景，红金暗金金属框，圆角，东方妖怪纹样，左侧圆角竖排名字牌，底部短技能铭牌，左上角预留三星区域，无角色，无场景，无文字，无技能图标，无首领标签，无巨大龙头，无满屏火焰雷电，整洁，可复用，基础3星首领卡质感但不过度传奇化
```

# 审核清单

生成 3 星卡牌 UI 或最终卡面时，必须检查：

- 左上角是否固定 3 颗星
- 名字是否竖排且完整可读
- 名字牌是否圆角
- 是否没有“首领”文字
- 是否没有“技能”文字
- 是否没有技能图标
- 是否没有无规则意义的右上角圆形徽章
- 技能区是否只显示一行效果文本
- 技能区是否没有过高
- 边框是否红金但不过度夸张
- 是否为未来 4 星、5 星保留升级空间
- UI 资产是否透明背景
- UI 资产是否无文字、无角色、无技能内容

# 当前确认模板

当前确认的 3 星基础模板方向：

```text
红金暗金边框
左上三星
左侧圆角竖排名字牌
底部短技能铭牌
无技能图标
无“技能”文案
无“首领”标签
无右上角属性徽章
不过度妖王化
```

该模板用于基础牌 3 星首领，不用于 4 星、5 星、收藏卡终极边框。
