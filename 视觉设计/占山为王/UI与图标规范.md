# 《夕妖：占山为王》UI与图标规范

## 文件定位

本文件定义《占山为王》卡牌与网页界面的图标、徽章、数字与状态视觉规范。

本文件不定义程序实现。

卡牌名称、星级、战力、技能以：

`游戏/卡牌游戏/占山为王/V1.2-基础卡表.md`

为唯一来源。

卡面是否展示某类信息，以：

`视觉设计/占山为王/卡牌框架规范.md`

为统一来源。

---

# UI目标

UI必须服务：

- 快速识别卡牌类型
- 快速识别星级
- 快速识别地盘归属
- 快速识别无主 / 已占领状态
- 快速识别法宝效果
- 手机端小图可读
- 实体卡牌印刷可读

说明：

妖怪战力只存在于规则与卡表中，不作为当前基础妖怪卡面的独立UI数字展示。

---

# 核心图标

必须建立以下图标组：

| 图标 | 用途 |
|---|---|
| 1星 | 小妖星级，也是基础结算识别 |
| 2星 | 精英星级，也是基础结算识别 |
| 3星 | 首领星级，也是基础结算识别 |
| 妖怪 | 妖怪卡类型 |
| 法宝 | 法宝卡类型 |
| 地盘 | 地盘卡类型 |
| 无主 | 无主地盘状态 |
| 已占领 | 已占领地盘状态 |
| 攻山 | 攻击行动 |
| 招兵 | 出牌驻守行动 |

基础妖怪卡不建立单独战力数字图标。

---

# 星级图标

星级图标应保持统一形状，只通过颜色与数量区分：

- 1星：青铜
- 2星：银铁
- 3星：暗金

星级是妖怪卡最重要的结算UI之一。

要求：

- 星级必须比普通装饰更醒目。
- 星级位置必须统一。
- 星级大小必须支持手机端小图阅读。
- 3星卡的三星可以适度放大，用于强化结算识别。

禁止每张卡使用不同星级图案。

---

# 妖怪卡战力展示

当前基础妖怪卡不单独显示战力数字。

说明：

- 战力数值以 `V1.2-基础卡表.md` 为规则来源。
- 当前基础卡面通过星级、名称与技能文本完成阅读。
- 不在卡面右上角增加战力数字。
- 不显示攻 / 守数值区。
- 不显示攻击力、防守力、攻3、守3等视觉元素。

若未来版本需要战力数字，应先更新规则与卡牌框架规范，不得仅在本文件单独新增。

---

# 阵营徽章

四大地盘阵营需要独立徽章：

| 阵营 | 徽章方向 |
|---|---|
| 白骨 | 骨纹 / 鬼火 |
| 火云 | 火焰 / 云纹 |
| 狮驼 | 狮首 / 妖城门 |
| 盘丝 | 蛛网 / 毒纹 |

阵营徽章只用于地盘阵营识别，不代表妖怪只能驻守该阵营。

妖怪卡不展示阵营徽章。

---

# 法宝功能图标

当前法宝功能：

| 功能 | 图标方向 |
|---|---|
| 收 | 吸纳漩涡 |
| 破 | 裂环 / 破法 |
| 抓 | 缠绕绳索 |
| 吹 | 风旋 / 扇形气流 |

法宝图标必须服务功能识别，不得加入规则中不存在的新功能。

---

# 状态图标

当前基础版只需要：

- 无主
- 已占领
- 出兵地
- 目标地盘

暂不设计复杂持续状态。

---

# UI资产化原则

卡牌UI应尽量资产化，避免每张卡重新生成一套不同UI。

建议固定资产包括：

- 1星边框
- 2星边框
- 3星边框
- 法宝边框
- 地盘边框
- 卡牌背面
- 名称栏
- 技能栏
- 星级图标
- 阵营徽章
- 法宝功能图标

角色、法宝、地盘插画可以分别生成，但基础UI应复用固定资产。

目标：

第1张卡与第52张卡的UI结构保持一致。

---

# 透明PNG导出建议

用于合成的UI素材建议导出为透明PNG。

要求：

- 无文字
- 无乱码
- 无多余图标
- 保留完整边框与装饰
- 角色插画与UI分层管理
- 名称与技能文本由后期排版或程序生成

---

# 禁止项

禁止：

- 图标风格混用现代科技UI
- 每张卡自定义一套星级图标
- 图标过小导致手机端不可读
- 阵营徽章暗示不存在的阵营加成
- 添加规则中不存在的状态图标
- 妖怪卡显示战力数字
- 妖怪卡显示攻击力、防守力、攻3、守3
- 妖怪卡显示阵营徽章
- 每次AI生成时重做一套不一致的基础UI
