# 《夕妖：占山为王》卡牌框架规范

## 文件定位

本文件定义《占山为王》实体卡与网页卡的版式框架。

本文件只定义视觉与信息层级，不定义规则内容。

卡牌名称、数量、技能、星级、战力以：

`游戏/卡牌游戏/占山为王/V1.2-基础卡表.md`

为唯一来源。

卡面短文案以：

`游戏/卡牌游戏/占山为王/V1.2-卡面技能文案.md`

为显示来源。

---

# 卡牌定位

《占山为王》卡牌定位为：欢乐聚会卡牌。

视觉目标：

- 一眼看懂
- 一眼认出角色
- 聚会场景快速阅读
- 收藏级但不复杂
- 适合实体打印与网页展示
- 具备商品级卡牌质感

参考方向：

- 三国杀式快速阅读
- 炉石式角色识别
- 皇室战争式立体卡牌感
- 魔兽兵团式角色冲击力
- 爆炸猫式聚会易读性

不采用重竞技卡牌或复杂战争桌游的信息密度。

---

# 商品级立体卡牌要求

当前卡牌UI必须从“资料卡布局”升级为“商品级收藏卡布局”。

必须具备：

- 厚度
- 金属感
- 浮雕感
- 层次感
- 景深
- 插画突破感
- 正式商品质感

卡面需要形成三层视觉结构：

1. 背景层：场景、妖云、火焰、山头、洞府、景深。
2. 主体层：妖怪、法宝或地盘主体，可适度突破边框。
3. UI层：星级、名称、技能卷轴、边框与角饰。

禁止平面资料卡风格。

禁止像 Word 表格、百科卡、档案卡、原型测试卡。

---

# 标准尺寸与印刷

实体卡建议尺寸：

- 63 × 88 mm
- 竖版
- 圆角
- 适配卡套

印刷文件必须预留：

- 3mm 出血
- 安全区内放置文字与关键图标
- 背景、花纹、边框必须延伸至出血区

禁止：

- 白边
- 断边
- 背景突然结束
- 关键文字贴近裁切线

网页展示可使用同一比例。

---

# 卡面信息层级

卡面必须清晰展示：

1. 星级或卡牌类型标识
2. 卡名
3. 插画主体
4. 简洁技能文本

妖怪卡不展示：

- 攻击力
- 防守力
- 攻 / 守数值区
- 妖怪阵营区
- 技能类型标签

说明：

- 战斗规则由星级与规则结算承担，不在卡面另设攻击力、防守力。
- 妖怪来源/归属只作为卡表中的世界观说明，不在卡面作为阵营规则展示。
- 所有首领技能默认为登场触发，卡面不显示“登场技”“主动技”“被动技”等技能名。

---

# 新版妖怪卡结构

妖怪卡采用插画主导型收藏卡结构。

从上到下：

1. 左上角星级区
2. 大面积插画区
3. 叠压式书法名称区
4. 底部卷轴 / 石碑式技能区
5. 四周立体金属边框与角饰

示意：

```text
┌────────────────────┐
│ ★★★                │
│                    │
│                    │
│      插画主体       │
│   可适度突破边框    │
│                    │
│ 金鼻白毛老鼠精      │
├────────────────────┤
│ 偷1只1星守军来守地盘 │
└────────────────────┘
```

比例建议：

- 插画区：70%~80%
- 名称区：10%~15%
- 技能区：10%~15%

---

# 星级位置

星级必须固定在左上角。

原因：

星级即战力，玩家需要频繁查看星级进行结算。

要求：

- 1星显示1颗星
- 2星显示2颗星
- 3星显示3颗星
- 星级使用金属浮雕大星
- 位置稳定，不得随卡牌变化
- 不得放到底部或名称下方作为纯装饰

---

# 妖怪名称留位规范

妖怪名称是卡面最高优先级文字之一。

名称栏必须按最长首领名预留空间：

`金鼻白毛老鼠精`

要求：

- 至少可完整容纳7个中文字。
- 不得遮挡、裁切或缩到不可读。
- 不得被花纹压住。
- 名称应是卡面最大或最醒目的文字之一。
- 长名优先横排完整显示；若必须换行，仍需保持清晰可读。
- 名称应具有书法感、浮雕感、压在插画上的商品卡效果。

---

# 技能文本规范

所有有技能的妖怪牌，默认都是出场时触发。

卡面不显示技能名。

不显示：

- 登场技
- 出场技
- 主动技
- 被动技
- 技能类型括号

技能区只显示效果文本。

技能文本来自：

`V1.2-卡面技能文案.md`

技能文本要求：

- 一句话
- 不加句号
- 不换行优先
- 玩家应能在3秒内读完
- 超过宽度时优先缩小字号，不得挤压插画主体

技能区视觉要求：

- 不做普通矩形备注栏
- 不做资料卡说明栏
- 采用卷轴、石碑、古牌匾或妖纹铭牌结构
- 与边框形成统一立体材质

---

# 字体方向

建议使用具备古意但清晰的中文标题字。

正文必须优先可读。

禁止使用难以辨认的书法字体作为技能正文。

名称字体可以更有风格，但必须完整可读。

名称建议具备：

- 书法笔触
- 金色或米白描边
- 阴影
- 浮雕厚度

---

# 边框与花纹系统

卡面四周必须填满连续花纹。

要求：

- 顶部有花纹
- 底部有花纹
- 左右有花纹
- 四角有完整花纹
- 花纹延伸至出血区
- 裁切后仍保持完整收藏卡感
- 边框具有金属铸造厚度
- 四角具有明显立体角饰

边框方向：

| 卡牌类型 | 边框方向 |
|---|---|
| 1星小妖 | 青铜边框 |
| 2星精英 | 银铁边框 |
| 3星首领 | 暗金边框 |
| 法宝 | 紫金神兵边框 |
| 地盘 | 阵营色边框 |

---

# 法宝卡结构

法宝卡同样采用插画主导型收藏卡结构。

要求：

- 法宝主体占画面70%~80%
- 法宝可悬浮于画面中央
- 法力旋涡、金光、紫气可突破边框
- 不显示星级
- 显示法宝类型标识
- 底部显示法宝效果短文案
- 使用紫金神兵边框

---

# 地盘卡结构

地盘卡保持竖版实体卡尺寸，但主体必须是场景。

要求：

- 不使用人物立绘模板
- 不使用妖怪卡布局
- 场景占画面80%左右
- 地名醒目
- 不显示星级
- 不显示技能栏
- 不显示复杂规则说明
- 可使用阵营徽记，但不做复杂阵营区

地盘也要有立体感：

- 山体、洞府、火焰、骨岭、妖云可形成景深
- 重要地标可以突破边框
- 不能像普通风景缩略图

---

# 可读性要求

所有卡牌缩小到手机屏幕宽度一半时，仍需识别：

- 卡名
- 卡牌类型
- 星级或法宝类型
- 是否有技能文本

---

# 禁止项

禁止：

- 出现攻击力、防守力、攻3、守3等区域
- 在妖怪卡上展示阵营区
- 显示“登场技”“主动技”“被动技”等技能名
- 复杂装饰压住卡名
- 插画盖住技能文字
- 星级位置不统一
- 不同卡牌随意换版式
- 为了收藏感牺牲规则阅读
- 未预留3mm出血导致印刷白边
- 平面UI
- 资料卡UI
- Word表格式布局
- 百科卡风格
- 档案卡风格
- 普通矩形备注栏
- 名称区像表格标题
- 技能区像说明备注
