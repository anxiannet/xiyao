# 游戏用棋子规范

本文件是《夕妖》战斗内棋子资源的唯一执行规范。

本文件只维护游戏用棋子与战斗内资源的生产标准，不新增世界观建筑、地形、角色设定或游戏规则元素。

```text
游戏用棋子 ≠ 展示用棋子 ≠ 宣传海报 ≠ 角色卡图
```

---

## 适用范围

适用于：

- 战斗地图中的英雄单位。
- 战斗地图中的非英雄单位。
- 移动、攻击、受击、待机等战斗方向资源。
- 进入前端游戏棋盘的棋子切图。
- 用于战斗内合成展示的角色本体资源。

不适用于：

- 角色详情页展示。
- 图鉴收藏展示。
- 抽卡 / 获得角色展示。
- 宣传封面。
- 收藏级手办渲染。
- 带 UI 面板的角色卡图。

以上展示类资产统一参考：

```text
视觉设计/棋子规范/展示用棋子规范.md
```

---

## 来源优先级

游戏用棋子必须遵循：

```text
世界观人物资料
↓
视觉与美术总规范
↓
阵营 / 小队视觉规范
↓
角色专项视觉规范
↓
游戏规则与地图规范
↓
本文件
↓
生成图 / 切图 / 前端资源
```

禁止用生成图反向修改人物设定、阵营设定、规则设定或地图设定。

---

## 核心目标

游戏用棋子的第一目标是战斗可读性。

优先级：

```text
规则识别 > 战术阅读 > 角色识别 > 阵营气质 > 收藏感 > 细节表现
```

说明：

- 游戏用棋子首先服务棋盘判断。
- 缩小后仍要能识别阵营、职业或战场功能。
- 不以宣传级细节、复杂材质或收藏级地台作为通过标准。
- 不允许为了展示好看而牺牲棋盘可读性。

---

## 统一镜头

战斗内所有资源必须使用同一镜头。

```text
Camera Type：Orthographic（正交投影）
Pitch：45°
Yaw：45°
FOV：不使用透视FOV
```

适用范围：

- 棋子。
- 地块。
- 当前规则确认的建筑或据点资源。
- 障碍物。
- 战斗场景装饰。

禁止：

- 地图45°、棋子35°。
- 棋子使用正面展示镜头。
- 棋子使用手办展示镜头。
- 地图与建筑使用不同镜头。
- 战斗资源与六边形地图透视不一致。
- 使用透视投影导致远处单位或地块缩小。

---

## 六边形地图方向体系

采用六边形地图。

统一使用 Dir1~Dir6 方向体系。

游戏逻辑、AI、技能方向、美术资源必须使用同一方向定义。

```text
Dir1：右前
Dir2：右
Dir3：右后
Dir4：左后
Dir5：左
Dir6：左前
```

生产时只维护3张基础源图：

```text
侧前
侧面
侧后
```

镜像后覆盖剩余3个方向。

```text
侧前 → 镜像侧前
侧面 → 镜像侧面
侧后 → 镜像侧后
```

不采用8方向资源。

禁止使用：

- 正前。
- 正后。
- 正前 / 侧面 / 正后三视图。
- 展示三视图。
- 宣传构图截图。

---

## 游戏棋子基础规格

标准：

```text
角色显示高度：200px
源图背景：#00FF00 纯绿幕
源图数量：3张基础源图
最终方向：Dir1~Dir6
镜头：正交投影 45°
底座：不在源图内，由程序统一挂载
```

移动端三档可读性目标：

| 显示高度 | 目标 |
|---:|---|
| 200px | 能识别具体角色 |
| 100px | 能识别职业 / 战场功能 |
| 50px | 能识别阵营 |

任一档失败，视为游戏用棋子不可用。

---

## 比例与造型

游戏用棋子允许使用轻度战棋化比例，但不得变成完全展示手办比例或纯头像 Q 版。

推荐：

```text
头身：约 3.5~5 头身
轮廓：清晰、紧凑、可剪影识别
动作：战斗待机姿态
重心：稳定站立，避免大幅外扩
```

允许：

- 适度放大头部与核心道具，提高小尺寸识别。
- 适度简化服装细节。
- 适度强化职业武器、尾巴、耳朵、披风、灯、杖等轮廓识别点。
- 使用低强度阵营色微光辅助识别。

禁止：

- 7头身以上展示手办比例直接进入战斗棋盘。
- 纯大头贴式 Q 版。
- 角色轮廓被长裙、翅膀、披风、特效完全糊成一团。
- 身体姿态过度扭曲导致方向不可读。
- 为了好看加入未确认的巨大机械翼、王座、月轮、展示地台或场景装置。

---

## 战斗棋子源图规则

游戏用战斗棋子源图只包含：

```text
角色本体
随身道具
必要的低强度角色识别微光
```

游戏用战斗棋子源图必须满足：

```text
纯绿幕背景 #00FF00
无地台
无底座
无圆盘
无石台
无展示座
无地面
无场景碎片
无地面投影
无外部阴影
无UI
无文字
无技能特效
无角色卡边框
无信息面板
```

说明：

- “无地台”不是“无地台外阴影”。
- “无地台”表示整个地台、底座、圆盘、石台、展示座都不存在。
- 角色脚下不得生成任何承托物。
- 角色不得站在圆形底座、发光法阵、石板、草地、六边形格或展示台上。
- 角色源图不得自带棋盘格、移动范围、攻击范围或 UI 标识。

正确生产链路：

```text
AI生成角色源图
↓
绿幕抠图
↓
只保留角色本体与随身道具
↓
导出透明 PNG / WebP
↓
导入游戏
↓
程序统一挂载底座 / 选中圈 / 状态圈 / 地块状态
```

---

## 程序底座规则

战斗中的底座不是棋子源图的一部分。

底座由程序或统一资源层挂载。

参考尺寸：

```text
角色高度：200px
程序底座直径：80px
六边形外接圆：90px
```

程序底座承担：

- 站位提示。
- 选中态。
- 阵营色。
- 状态圈。
- 可交互高亮。
- 移动 / 攻击范围的视觉连接。

棋子源图不得承担这些职责。

---

## 棋盘占格规则

游戏用棋子必须清楚表达“站在一个格子上”。

要求：

- 主体视觉重心位于当前六边形格中心。
- 脚部或落点不得明显偏离格心。
- 角色外轮廓可以略微超出格子，但不得遮挡相邻格核心信息。
- 武器、披风、尾巴、发丝可轻微越界，但不得影响点击判断和移动范围阅读。

禁止：

- 大范围横向武器遮挡邻格。
- 大面积特效覆盖邻格。
- 角色姿态导致落点不明确。
- 底部自带阴影造成站位误判。

---

## 角色识别原则

每名角色必须拥有唯一识别核心。

角色识别核心必须来自对应角色专项视觉规范，不在本文件重复维护。

参考来源：

```text
视觉设计/角色规范/*.md
```

识别优先级：

```text
轮廓 > 核心道具 > 职业武器 > 阵营色 > 服装细节 > 小纹样
```

游戏用棋子不得依赖以下内容完成识别：

- 文字标签。
- UI 名牌。
- 复杂服装花纹。
- 角色卡背景。
- 宣传图构图。
- 大面积技能特效。

---

## 职业 / 功能可读性

棋子应通过轮廓和随身道具表达大致战场功能。

常见识别方式：

| 功能倾向 | 视觉重点 |
|---|---|
| 近战 | 短兵器、前倾姿态、紧凑攻击轮廓 |
| 远程 | 弓、枪、法器、投射道具轮廓 |
| 辅助 | 灯、杖、铃、旗、治疗或引导类道具 |
| 坦克 | 大盾、重甲、宽体块面 |
| 刺客 | 轻装、短刃、低重心、快速姿态 |
| 控制 | 法阵类道具、小型控制器物、明确施法手势 |

以上仅作为视觉表达方法，不新增职业规则。

---

## 特效限制

游戏用棋子允许低强度识别微光。

允许：

- 武器或核心道具的微弱发光。
- 阵营色边缘微光。
- 不遮挡身体轮廓的少量粒子。
- 待机状态下非常轻量的能量提示。

禁止：

- 技能释放特效常驻在源图中。
- 大面积闪电、火焰、烟雾、晶体爆发。
- 环绕身体的大型光环。
- 超出角色高度或宽度太多的特效拖尾。
- 用特效代替角色本体识别。

技能表现应作为独立战斗特效资源处理，不进入棋子本体源图。

---

## 风格边界

游戏用棋子可以吸收幻想二次元、轻机甲、战棋模型感，但必须服务游戏可读性。

允许：

- 轻度幻想化服装。
- 轻度机甲装甲部件。
- 低强度发光材料。
- 战棋小模型质感。
- 适合移动端识别的简化比例。

限制：

- 机甲部件不得覆盖角色核心识别。
- 猫耳、狐耳、尾巴、武器等角色识别元素不得被装甲替换为新设定。
- 不得使用其他商业游戏的 Logo、UI、标题或可识别专属版式。
- 不得把“某游戏风格”直接作为项目规范名称。

---

## 游戏内合成展示

前端可以在游戏内用程序合成以下效果：

- 六边形底座。
- 阵营色光圈。
- 选中高亮。
- 行动状态。
- 受击闪白。
- 血条、能量条、状态图标。
- 移动范围与攻击范围。

这些属于游戏 UI / 程序表现，不属于棋子源图。

棋子源图不得预先烘焙这些元素，避免后续状态叠加冲突。

## 地块资源

所有地图地块统一：

```text
Camera Type：Orthographic（正交投影）
Pitch：45°
Yaw：45°
```

包括：

- 草地。
- 石地。
- 道路。
- 河流。
- 桥梁。
- 森林。
- 山地。

禁止单独改变地块镜头。

---

## 建筑与据点资源

战斗内建筑与据点资源统一：

```text
Camera Type：Orthographic（正交投影）
Pitch：45°
Yaw：45°
```

当前 MVP 明确维护：

- 中央据点。
- 边缘据点。
- 可作为普通装饰的旧驿站残留。
- 可作为普通装饰的残墙或路标。

未在世界观或规则中确认的建筑类型，不得作为核心战斗资源加入本规范。

## 生成提示词模板

用于生成游戏用棋子源图时，可按以下结构写提示词：

```text
《夕妖》游戏用战棋棋子，单个角色，正交投影，45度俯视角，六边形战棋游戏可用，轻度战棋化比例，角色高度约200px，小尺寸可读，清晰剪影，侧前方向 / 侧面方向 / 侧后方向，纯绿幕背景#00FF00，只保留角色本体和随身道具，无底座，无地台，无圆盘，无地面，无场景，无UI，无文字，无技能特效，无外部阴影。
```

角色专项内容必须从对应角色视觉规范补入，例如：

```text
角色名称：按角色规范
阵营 / 小队色：按阵营或小队规范
核心识别道具：按角色规范
职业或战场功能视觉倾向：按规则与角色规范
```

禁止在提示词中加入：

- 展示级棋子。
- 收藏级手办。
- 巨型王座。
- 圆形展示底座。
- 宣传海报。
- 角色卡 UI。
- 非项目确认的世界观元素。

## 审核检查清单

进入游戏前必须检查：

- 是否为正交投影45°。
- 是否只有3张基础源图。
- 是否能镜像覆盖6方向。
- 是否纯绿幕或已正确抠为透明背景。
- 是否无地台、无底座、无圆盘、无地面。
- 是否无 UI、无文字、无角色卡信息面板。
- 是否无技能特效和大面积粒子。
- 是否在200px、100px、50px三档可读。
- 是否符合角色专项视觉规范。
- 是否未新增世界观、规则或地图设定。

---

## 禁止事项

- 使用8方向资源作为默认生产标准。
- 使用正前 / 侧面 / 正后三视图作为战斗方向资源。
- 使用展示三视图作为战斗方向资源。
- 地图与棋子使用不同镜头。
- 展示资源覆盖游戏规范。
- 为展示效果牺牲战棋可读性。
- 战斗棋子源图包含地台、底座、圆盘、石台、展示座或地面。
- 战斗棋子源图包含角色卡 UI、信息面板、文字说明或宣传版式。
- 将展示手办图误用为游戏战斗棋子源图。
- 在游戏用棋子规范中新增未确认世界观建筑、地形、角色事实或规则元素。
