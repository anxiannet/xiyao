# 棋子审核标准

本文件用于审核棋子是否符合：

- `视觉设计/棋子规范/游戏用棋子规范.md`
- `视觉设计/棋子规范/展示用棋子规范.md`
- `视觉设计/角色品质等级规范.md`

审核时必须先确认资产用途。

```text
游戏用棋子 ≠ 展示用棋子
```

禁止用同一套标准同时审核战斗资源和展示资源。

---

## 审核入口

| 资产用途 | 使用标准 |
|---|---|
| 战斗地图单位 | 游戏用棋子审核 |
| 移动 / 攻击 / 受击方向图 | 游戏用棋子审核 |
| 角色详情页展示 | 展示用棋子审核 |
| 图鉴 / 收藏模型展示 | 展示用棋子审核 |
| 宣传单体棋子 | 展示用棋子审核 |

---

# 一、游戏用棋子审核

游戏用棋子的核心原则：

```text
规则识别 → 战术阅读 → 角色识别 → 收藏感 → 细节表现
```

禁止使用花纹数量作为主要评分依据。

---

## 1. 第一眼识别测试（必过）

审核人观察棋子2秒后回答：

- 属于什么阵营。
- 大概是什么职业或战场功能。
- 是否为重要角色。

无法回答则不通过。

---

## 2. 移动端三档可读性（必过）

| 显示高度 | 审核目标 |
|---:|---|
| 200px | 能识别具体角色 |
| 100px | 能识别职业 / 战场功能 |
| 50px | 能识别阵营 |

任一档失败，退回调整。

---

## 3. 轮廓识别测试（30分）

图片缩小为100px，仅保留剪影。

要求：

- 能识别角色类型。
- 主要识别点来自轮廓。
- 不依赖花纹和小配件。

评分：

- 25~30：轮廓唯一。
- 15~24：轮廓一般。
- 0~14：轮廓混乱。

---

## 4. 阵营识别（20分）

图片缩小至50px。

要求：

- 能通过主色与核心元素识别阵营。
- 不依赖文字标签识别阵营。

---

## 5. 职业 / 功能识别（20分）

图片缩小至100px。

要求：

- 能通过武器或核心道具判断功能定位。
- 不依赖复杂服装花纹判断功能。

---

## 6. 核心道具测试（10分）

每个角色必须拥有1~2个唯一识别道具。

遮挡角色主体后，仅看道具仍应具备识别价值。

---

## 7. 棋盘可读性（10分）

要求：

- 不遮挡相邻格子。
- 不遮挡路线。
- 不使用超大特效。
- 不使用超复杂底座。
- 角色朝向清晰。

---

## 8. 资源规范检查（必过）

2D游戏用棋子必须符合：

- 基础源图为3张：侧前、侧面、侧后。
- 镜像后覆盖6方向。
- 成品资源以Dir1~Dir6进入游戏。
- 源图使用绿幕背景。
- 源图不含地台。
- 源图不含UI、文字、场景。
- 源图不含技能特效。

不通过项：

- 使用8方向图作为六边形地图默认资源。
- 使用“正前 / 侧面 / 正后”三面图冒充战斗方向资源。
- 手工维护镜像派生图并与基础源图产生设定差异。

---

## 9. 材质表现（10分）

考察：

- 布料。
- 金属。
- 皮革。
- 发光材质。

游戏用棋子只要求基础品质感，不以宣传级材质为通过门槛。

---

## 游戏用等级判定

| 分数 | 判定 |
|---:|---|
| 90~100 | L3收藏棋子，可由游戏资产派生展示资产 |
| 75~89 | L2游戏棋子，当前MVP主标准 |
| 60~74 | L1测试棋子，只可用于验证 |
| 60以下 | 退回重做 |

---

# 二、展示用棋子审核

展示用棋子的核心原则：

```text
角色识别 → 收藏感 → 阵营气质 → 材质表现 → 战棋可读性
```

展示用棋子不得替代游戏用棋子。

---

## 1. 角色识别（必过）

审核人观察展示图2秒后，应能回答：

- 这是哪个角色。
- 属于哪个小队或阵营。
- 核心道具是什么。

无法回答则不通过。

---

## 2. 收藏感审核

允许：

- 精细服装。
- 独特武器。
- 专属地台。
- 轻量展示特效。
- 更强材质表现。

禁止：

- 纯功能图标化设计。
- 用地台或特效抢走角色主体。
- 用展示图反向修改人物设定。

---

## 3. 展示镜头审核

允许：

- 15°~25°手办展示视角。
- 35°战斗棋子参考视角。
- 正前 / 侧面 / 背面三视图。

必须标注用途：

- 展示图。
- 建模参考。
- 战斗棋子参考。

禁止：

- 将展示三视图当作六边形战斗方向资源。
- 将宣传构图当作战斗地图单位资源。

---

## 4. 展示地台审核

允许：

- 狐纹阵。
- 狐爪纹。
- 狐火法阵。
- 追猎纹。
- 阵营低矮装饰。

禁止：

- 地台喧宾夺主。
- 地台加入未确认世界观设定。
- 展示地台直接覆盖战斗统一底座标准。

---

## 5. 展示特效审核

允许：

- 核心道具发光。
- 阵营色微光。
- 地台低强度光痕。
- 少量粒子。

禁止：

- 满屏特效。
- 超大烟雾。
- 大面积爆炸。
- 遮挡角色主体。
- 用特效代替角色识别。

---

# 三、苏绫参考案例

当前苏绫棋子核心识别：

- 狐耳。
- 旅行斗篷。
- 引路狐灯。
- 引路杖。
- 青丘配色。

轮廓识别优秀。

核心识别依赖：

```text
狐灯 + 引路杖
```

审核注意：

- 游戏用苏绫必须保证狐灯和引路杖在至少主要方向中可读。
- 展示用苏绫可以强化脸部、服装材质和地台收藏感。
- 展示用苏绫不得替代战斗方向资源。

---

## 退回重做条件

出现以下任一情况，直接退回：

- 与人物志事实冲突。
- 与小队视觉规范冲突。
- 无法通过200px / 100px / 50px三档可读性。
- 游戏用资源方向不符合Dir体系。
- 游戏用资源使用8方向作为默认生产标准。
- 游戏用资源使用“正前 / 侧面 / 正后”三视图冒充战斗方向。
- 展示图反向修改人物设定或游戏规则。
- 特效、地台或场景遮挡角色主体。
