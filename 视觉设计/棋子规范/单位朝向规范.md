# 单位朝向规范

## 适用范围

适用于《夕妖》所有棋盘单位。

包括：

- MVP阶段2D六方向棋子
- 后续手机App阶段3D模型
- 单位移动
- 单位攻击
- 技能方向
- AI决策朝向

---

## 核心原则

《夕妖》采用六角格真实方向体系。

棋盘方向 = 模型方向。

所有单位朝向、资源命名、技能方向、AI方向均使用同一套Dir体系。

禁止在规则、资源或动画中使用前、后、左、右作为正式方向。

---

## 六方向定义

```text
Dir1 = 东
Dir2 = 东南
Dir3 = 西南
Dir4 = 西
Dir5 = 西北
Dir6 = 东北
```

示意：

```text
        Dir6 东北
   Dir5 西北      Dir1 东

   Dir4 西        Dir2 东南
        Dir3 西南
```

---

## MVP阶段：2D六方向图

MVP阶段使用2D六方向棋子图。

每个英雄单位至少提供6张待机方向图：

```text
角色名_Dir1.png
角色名_Dir2.png
角色名_Dir3.png
角色名_Dir4.png
角色名_Dir5.png
角色名_Dir6.png
```

示例：

```text
阿照_Dir1.png = 阿照面向东
阿照_Dir2.png = 阿照面向东南
阿照_Dir3.png = 阿照面向西南
阿照_Dir4.png = 阿照面向西
阿照_Dir5.png = 阿照面向西北
阿照_Dir6.png = 阿照面向东北
```

---

## 后续App阶段：3D模型

手机App阶段可改用3D模型。

3D模型不再需要6张方向图。

仍然保留同一套Dir体系。

Dir只从图片切换变为模型旋转。

示例：

```text
Dir1 = rotation 0°
Dir2 = rotation 60°
Dir3 = rotation 120°
Dir4 = rotation 180°
Dir5 = rotation 240°
Dir6 = rotation 300°
```

规则层、AI层、技能层不需要因为2D升级3D而改变方向定义。

---

## 数据结构建议

单位数据中只记录Dir编号。

示例：

```json
{
  "unit_id": "azhao",
  "position": {
    "q": 5,
    "r": 3
  },
  "dir": 2
}
```

含义：

```text
阿照位于 q=5 r=3
当前面向 Dir2
即东南方向
```

---

## 技能方向规则

技能描述中的“前方”统一解释为当前Dir方向。

示例：

```text
当前Dir = Dir2
前方 = 东南
```

```text
当前Dir = Dir5
前方 = 西北
```

禁止技能单独定义另一套方向系统。

---

## AI方向规则

AI决策只输出Dir编号。

示例：

```text
AI决定阿照移动后面向 Dir3
```

MVP阶段表现为：

```text
显示 阿照_Dir3.png
```

3D阶段表现为：

```text
阿照模型旋转至 Dir3 对应角度
```

---

## 资源生成要求

生成六方向图时，必须以Dir体系命名与排列。

禁止使用：

- front
- back
- left
- right
- 前
- 后
- 左
- 右

允许使用：

- Dir1
- Dir2
- Dir3
- Dir4
- Dir5
- Dir6

---

## 视觉设计要求

所有角色在六方向中必须保持：

- 同一比例
- 同一地台尺寸
- 同一光照方向
- 同一镜头角度
- 同一透明背景

角色核心识别物必须在至少4个方向中清晰可见。

示例：

- 苏绫：狐印
- 阿照：狐面
- 青萝：狐火灯
- 琉尾：狐尾刃

---

## 禁止事项

禁止MVP阶段使用一套方向，3D阶段再改另一套方向。

禁止棋盘逻辑使用六角方向，而图片资源使用前后左右方向。

禁止技能方向、AI方向、美术资源方向各自独立。
