## 一、地图定位

青丘谷不是仙境秘境，也不是单纯狐狸森林。

青丘谷地图视觉定位：

```text
青丘边境古路
裂隙监测区
青丘使团驻守区
山谷通行节点
```

关键词：

```text
古路
山谷
溪流
断崖
狐灯
裂隙
路标
旅人
巡路
黄昏残痕
```

禁止方向：

```text
仙侠秘境
童话森林
魔法学院
宫廷花园
赛博地形
暗黑魔域
```

---

## 二、地图规模

青丘谷作为标准自由对战地图，视觉上应支持：

```text
中央据点 ×1
边缘据点 ×4
黄昏裂隙 ×2
掩影区 ×4
```

地图应以完整场景方式制作，再由程序叠加六边形网格。

不要以大量独立六边形插画直接拼接成地图。

---

## 三、整体构图

推荐整体布局：

```text
        边缘据点

    掩影      掩影

边缘据点   中央据点   边缘据点

    裂隙      裂隙

    掩影      掩影

        边缘据点
```

构图目标：

- 中央据点一眼可见。
- 四个边缘据点形成战略边界。
- 两处黄昏裂隙形成危险路线。
- 四处掩影形成绕行与防御空间。
- 主路连接所有据点，但不能像棋盘格硬铺。

---

## 四、地表比例

### 草地

建议占比：

```text
55%~65%
```

表现：

- 明亮黄绿色山谷草地。
- 可见少量白花与低草。
- 细节密度中等，不堆料。
- 白天手机屏幕可清楚辨识。

### 古路 / 土路

建议占比：

```text
15%~25%
```

表现：

- 黄土旧路。
- 轻微车辙与磨损。
- 主路自然穿过多个格子。
- 不按每格硬切断。

### 石板遗迹

建议占比：

```text
8%~12%
```

表现：

- 旧石板。
- 古驿站残留。
- 据点周围更密集。
- 颜色偏暖灰，不压暗地图。

### 水系 / 崖边

建议占比：

```text
5%~10%
```

表现：

- 浅溪。
- 小瀑布。
- 山谷边缘水流。
- 只作为边缘景观，不干扰规则地块判断。

## 五、中央据点

中央据点视觉定位：

```text
青丘旧驿站核心节点
古路汇合处
地图战术中心
```

建议结构：

```text
低矮圆形石台
+
旧路标 / 旗帜
+
狐纹或路线纹
+
周边石板连接主路
```

要求：

- 高于普通地表，但不能像神殿。
- 不遮挡相邻格。
- 不画死归属颜色。
- 青丘、天门、中立归属由程序层表现。

禁止：

- 做成王座。
- 做成大型祭坛神殿。
- 做成巨大建筑。
- 使用满屏发光特效。

---

## 六、边缘据点

边缘据点视觉定位：

```text
边境路标
巡路节点
小型战略目标
```

建议结构：

```text
低矮石基
+
旗帜 / 路标
+
小型地面符纹
```

要求：

- 明显高于普通格。
- 明显低于中央据点。
- 只强调单格，不扩散到一圈。
- 不遮挡棋子与相邻格判断。

数量：

```text
4个
```

---

## 七、黄昏裂隙

数量：

```text
2处
```

视觉定位：

```text
夕子聚集形成的黄昏伤口
危险但可进入
青丘狐火来源点
```

表现：

- 地面微光裂纹。
- 夕照红、琥珀橙、暮紫微光。
- 裂纹深处暗红或暮紫阴影。
- 少量狐火余烬。

禁止：

- 蓝色裂缝。
- 绿色鬼火裂缝。
- 科幻能量裂缝。
- 立起来的传送门。
- 深坑、大裂谷、障碍物。

说明：

黄昏裂隙可以视觉上跨过多个地表区域，但实际规则触发格由程序或关卡数据决定。

---

## 八、掩影区

数量：

```text
4处
```

视觉定位：

```text
自然遮蔽
防御地形
山谷阴影
```

表现：

- 古树阴影。
- 残墙阴影。
- 岩体投影。
- 边缘柔和的暗绿区域。

要求：

- 看起来是可进入地面。
- 不能像黑雾。
- 不能像实体障碍。
- 不阻挡玩家理解路线。

---

## 九、青丘元素

青丘谷可以出现以下元素：

```text
狐灯
旧木路牌
风蚀路标
药草
小石堆
旧旗帜
巡路用小灯
```

使用原则：

- 只做世界观氛围，不承担规则。
- 不遮挡棋子。
- 不遮挡六边形格。
- 不要密集堆满地图。

狐灯颜色：

```text
夕照红
琥珀橙
暮金
```

禁止使用默认青蓝狐火或绿色鬼火。

---

## 十、整体色彩

地图整体要求：

```text
明亮
鲜艳
高对比
白天可读
有质感
不过度堆细节
```

推荐色彩方向：

| 类型 | 色彩方向 |
|---|---|
| 草地 | 黄绿色、暖绿 |
| 土路 | 黄土色、风沙棕 |
| 石板 | 暖灰、旧石色 |
| 水系 | 清亮青蓝，仅作环境水色 |
| 黄昏裂隙 | 夕照红、琥珀橙、暮紫 |
| 青丘灯火 | 夕照红、暮金 |
| 天门元素 | 执法白金、冷灰蓝 |

注意：

- 水系可以使用清亮青蓝，但不得与狐火、黄昏裂隙混淆。
- 狐火和黄昏裂隙不使用青蓝作为默认方向。

---

## 十一、光照与材质

统一光源：

```text
左上光
```

要求：

- 所有据点、石头、残墙、树木、地表阴影方向一致。
- 草地、石板、土路使用同一色温。
- 地图整体为明亮日光，不做阴暗夜景。
- 掩影可以暗，但不能黑。

材质要求：

- 草地柔和。
- 石板有体积感。
- 土路有磨损感。
- 据点有旧石与金属细节。
- 裂隙有发光体积，但不遮挡地形。

---

## 十二、完整地图底图要求

青丘谷完整地图底图必须：

- 无 UI。
- 无文字。
- 无角色。
- 无棋子。
- 无六边形网格。
- 不画死选中框、移动范围、攻击范围。
- 地表自然连续。
- 预留中央据点、四个边缘据点、两处裂隙、四处掩影位置。

程序层再叠加：

```text
六边形网格
据点归属
占领范围
移动范围
攻击范围
角色棋子
```

## 十四、禁止事项

- 不得生成 UI。
- 不得生成文字。
- 不得生成角色。
- 不得生成六边形网格到底图里。
- 不得加入规则文档没有的核心地形。
- 不得把青丘谷画成纯仙境或童话森林。
- 不得把黄昏裂隙画成蓝色科幻裂缝。
- 不得把掩影画成黑雾。
- 不得把中央据点画成大型神殿。
- 不得让地图看起来像许多六边形插画拼接。
