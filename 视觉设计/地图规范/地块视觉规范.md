# 地块视觉规范

本文件是《夕妖》战斗地图地块的视觉执行规范。

本文件只负责视觉表现，不新增地形规则。

地形规则唯一来源：

```text
游戏/游戏设定/地形系统.md
```

地图资源制作通用规范：

```text
视觉设计/地图规范/地图资源规范.md
```

---

## 视觉总原则

地块视觉优先级：

```text
规则识别
↓
战术阅读
↓
地块氛围
↓
装饰细节
```

所有地块必须满足：

- 手机端一眼可读。
- 六边形格边界清晰。
- 不遮挡棋子底座。
- 不遮挡移动范围、攻击范围、占领范围。
- 不用装饰物替代规则表现。

统一镜头：

```text
Camera Type：Orthographic（正交投影）
Pitch：45°
Yaw：45°
Hex外接圆：90px
```

---

## 地块资源分类

地块视觉分为四类：

```text
基础地形
据点地形
地块状态
程序提示层
```

说明：

- 基础地形决定地块本体。
- 据点地形属于地形，但拥有归属状态。
- 地块状态是附着在格子上的持续效果，不改变地形本体。
- 程序提示层用于选中、移动范围、攻击范围、占领范围。

---

# 一、基础地形视觉

## plain 普通格

规则：无特殊规则。

视觉定位：

```text
默认地面
低信息密度
不抢角色
```

建议表现：

- 浅草地。
- 浅土路。
- 浅石地。
- 低饱和、低对比。
- 不加明显图标。
- 不加特殊光效。
- 不加高亮边框。

生成要求：

- 使用无缝纹理。
- 不带六边形粗边。
- 不带强阴影。
- 不带独立装饰物。

---

## high_ground 高台

规则：远程攻击从高台发起时 RNG +1。

视觉定位：

```text
射程优势地形
需要明显看出“高”
```

建议表现：

- 低矮抬升平台。
- 石层、台阶、坡面。
- 顶部保留明确可站立平面。
- 高度明显，但不遮挡棋子底座。

视觉关键词：

```text
抬升
台阶
石层
坡面
可站立
```

禁止：

- 做成不可进入的山。
- 高度过高导致像障碍物。
- 平台边缘遮挡相邻格。

---

## cover_shadow 掩影

规则：防御方 DEF_DICE +1；假身在此生成持续 +1 回合。

视觉定位：

```text
防御 / 隐蔽地形
```

建议表现：

- 半透明阴影纹。
- 低矮树影、墙影、残垣影。
- 地表有暗色遮蔽感。
- 不做实体墙，不阻挡视线。

颜色建议：

```text
墨蓝灰 #2A313B
低透明度
边缘柔和
```

禁止：

- 画成实体障碍。
- 画成高墙。
- 让玩家误以为阻挡移动或视线。

---

## dusk_rift 黄昏裂隙

规则：青丘单位经过时收集1点狐火。

视觉定位：

```text
青丘相关资源地形
狐火来源点
```

建议表现：

- 地面裂隙。
- 裂隙中透出狐火青光。
- 少量流动光痕。
- 裂隙不抬高，不阻挡移动，不阻挡视线。

颜色建议：

```text
裂隙暗部：墨蓝灰 #2A313B
能量光：狐火青 #4FA3A5
边缘暖色：暮金 #C9A86A
```

禁止：

- 做成深坑障碍。
- 做成大裂谷。
- 光效遮挡棋子脚下状态。

---

## obstacle 障碍

规则：不可进入；阻挡移动和直线视野。

视觉定位：

```text
最明确的阻挡地形
```

建议表现：

- 石块。
- 断墙。
- 树干。
- 残骸。
- 封锁物。

视觉关键词：

```text
实体
不可进入
阻挡
高轮廓
```

要求：

- 轮廓必须高于普通地形。
- 占格边界明确。
- 必须明显区别于掩影。

禁止：

- 做得太低，像普通装饰。
- 只用阴影表现。
- 与掩影视觉混淆。

---

# 二、据点地形视觉

## central_objective 中央据点

规则：据点范围为中心格及相邻1圈格；归属方在回合结算时 +2 气运。

视觉定位：

```text
地图核心目标
最高规则识别度
```

建议表现：

- 中心格使用明显祭坛、阵纹、台基。
- 外圈1圈格使用低强度环形纹理提示范围。
- 中心格比普通格更亮，但不能遮挡棋子。
- 据点归属通过程序色层表现，不画死在底图。

颜色建议：

```text
无归属：执法白金 #D8D2BF
青丘归属：狐火青 #4FA3A5
天门归属：暮金 #C9A86A
```

表现层级：

```text
地块底图：据点结构
程序层：归属色
程序层：占领范围提示
```

禁止：

- 把青丘 / 天门归属直接画死在基础图。
- 中央据点做成大建筑遮挡周围格。
- 用满屏光效表现据点。

---

## edge_objective 边缘据点

规则：据点范围为中心格；归属方在回合结算时 +1 气运。

视觉定位：

```text
小型目标点
低于中央据点
高于普通格
```

建议表现：

- 小型石台。
- 标记柱。
- 地面符纹。
- 只强调当前格，不扩散到相邻格。
- 体积低矮，避免误判为障碍。

与中央据点区别：

```text
中央据点：中心 + 外圈范围感
边缘据点：单格目标感
```

---

## 据点归属视觉

据点拥有归属状态。

建议使用程序层表现归属：

| 归属 | 视觉 |
|---|---|
| 无归属 | 白金低亮符纹 |
| 青丘 | 狐火青呼吸光 |
| 天门 | 暮金 / 执法白金稳定光 |
| 争夺中 | 低频闪烁边线，不改变底图 |

禁止：

```text
在地块底图中画死归属
```

原因：

```text
归属会变化，底图不应承担状态表达
```

# 三、地块状态视觉

地块状态不是基础地形，只是附着在地图格子上的持续效果。

状态必须做成：

```text
程序叠加层
```

不得画死在地块底图里。

---

## 狐火残留

规则：青丘单位进入带有狐火残留状态的地块：收集1点狐火；敌方单位进入获得迷踪；不阻挡移动和视线；不能生成在障碍格。

视觉定位：

```text
青丘留下的可交互地块状态
```

建议表现：

- 地面上方低透明狐火纹。
- 小范围狐火青粒子。
- 六边形内侧轻微流光。
- 不改变地形原本形状。

颜色：

```text
狐火青 #4FA3A5
少量暮金 #C9A86A
```

要求：

- 可见，但不能盖住地形类型。
- 不能盖住棋子底座。
- 不能像黄昏裂隙本体。

---

## 勘验区

规则：指定1格及相邻格获得勘验区状态；持续到回合结束；带有勘验区状态的狐火残留、黄昏裂隙、掩影效果失效；不阻挡移动和视线。

视觉定位：

```text
天门 / 调查 / 压制性规则区域
```

建议表现：

- 几何扫描纹。
- 细线六边形边框。
- 执法白金色低透明覆盖。
- 可叠加在狐火残留、黄昏裂隙、掩影上，表示其效果被压制但没有移除。

颜色：

```text
执法白金 #D8D2BF
墨蓝灰 #2A313B
```

表现要求：

```text
状态压制感
规则清晰
不能像障碍
不能像占领范围
```

# 四、程序提示层

程序提示层不属于地块本体。

包括：

- 选中框。
- 移动范围。
- 攻击范围。
- 占领范围。
- 据点归属色。
- 勘验区覆盖层。
- 狐火残留覆盖层。

推荐层级：

```text
Layer 1：基础地面纹理
Layer 2：基础地形结构
Layer 3：地块状态
Layer 4：据点归属
Layer 5：程序高亮
Layer 6：棋子
Layer 7：技能预览 / 攻击范围
```

---

# 五、精灵图生产规范

地块精灵图用于生产和拆分资源，不是游戏最终整图。

精灵图应按以下分类排列：

```text
基础地形地块
据点地形
地块状态
占领范围提示
障碍物
装饰物
```

要求：

- 绿幕背景 `#00FF00`。
- 无UI。
- 无角色。
- 无宣传特效。
- 文字标签仅允许出现在参考图，不允许进入最终资源切片。
- 地块之间留足切割间距。
- 每个资源保持统一正交45°视角。

# 七、禁止事项

- 新增地形规则。
- 新增地形名称作为规则本体。
- 用视觉表现覆盖规则设定。
- 把地块状态画死进基础地形。
- 把据点归属画死进基础地形。
- 让掩影视觉像障碍。
- 让黄昏裂隙视觉像不可进入深坑。
- 让高台视觉像不可进入山体。
- 让装饰物遮挡路线判断。
- 使用透视投影导致远处地块缩小。
