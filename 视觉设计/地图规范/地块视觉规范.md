# 地块视觉规范

本文件是《夕妖》战斗地图地块的视觉执行规范。

本文件只负责视觉表现，不新增地形规则。

地形规则唯一来源：

```text
游戏/游戏设定/地形系统.md
```

地图资源制作通用规范：

```text
视觉设计/地图规范/地图资源规范.md
```

黄昏裂隙专项视觉规范：

```text
视觉设计/地图规范/黄昏裂隙视觉规范.md
```

狐火与夕子专项视觉规范：

```text
视觉设计/特效规范/狐火与夕子视觉规范.md
```

---

## 一、视觉总原则

地块视觉优先级：

```text
规则识别
↓
战术阅读
↓
整体地图感
↓
地块氛围
↓
装饰细节
```

所有地块必须满足：

- 手机端一眼可读。
- 不遮挡棋子底座。
- 不遮挡移动范围、攻击范围、占领范围。
- 不用装饰物替代规则表现。
- 不让单格像独立插画小岛。
- 拼合后必须像一张完整地图。

统一镜头：

```text
Camera Type：Orthographic（正交投影）
Pitch：45°
Yaw：45°
Hex外接圆：90px
统一光源：左上
整体色调：明亮、鲜艳、高对比、白天可读
```

---

## 二、资源分层

地块视觉分为：

```text
地表底图 / 无缝纹理
特殊地形覆盖层
据点资源层
地块状态层
程序提示层
装饰物层
```

说明：

- 地表底图负责整体地图自然过渡。
- 特殊地形覆盖层负责黄昏裂隙、掩影、高台等规则识别。
- 据点资源属于规则地形，但归属状态不得画死。
- 地块状态是附着在格子上的持续效果，不改变地形本体。
- 程序提示层用于选中、移动范围、攻击范围、占领范围。
- 装饰物不承担规则表达。

禁止把所有信息都画进单个六边形地块图。

---

## 三、整体地图感规范

为避免拼接感，地块资源必须遵循：

```text
完整地图优先
无缝纹理次之
独立六边形插画最后使用
```

完整地图底图中：

- 草地、土路、石板、溪流自然过渡。
- 一条路可以连续穿过多个格子。
- 黄昏裂隙可以跨越地表，但规则触发格由程序标记。
- 掩影可以来自树冠、残墙、岩体，但规则范围由程序标记。
- 六边形网格由程序统一绘制。

独立地块图仅用于资源生产参考，不作为默认拼图方式。

# 四、基础地形视觉

## plain 普通格

规则：无特殊规则。

视觉定位：

```text
默认地面
低信息密度
不抢角色
承接整体地图
```

建议表现：

- 明亮浅草地。
- 黄绿色山谷地表。
- 黄土旧路。
- 浅灰旧石板。
- 小范围泥土露出。

生成要求：

- 优先使用无缝纹理或完整底图。
- 不带六边形粗边。
- 不带强阴影。
- 不带固定装饰物。
- 不形成单格中心构图。

禁止：

- 荧光绿草坪。
- 灰暗写实泥地。
- 每格独立小岛感。
- 每格自带厚边框。

---

## high_ground 高台

规则：远程攻击从高台发起时 RNG +1。

视觉定位：

```text
射程优势地形
明显看出“高”
但仍可进入
```

建议表现：

- 低矮抬升平台。
- 岩层、台阶、坡面。
- 顶部保留明确可站立平面。
- 与周边草地或石路自然衔接。

高度建议：

```text
视觉高度：约16px~24px
```

禁止：

- 做成不可进入的山。
- 高度过高导致像障碍物。
- 平台边缘遮挡相邻格。
- 高台像独立小岛插在地图上。

---

## cover_shadow 掩影

规则：防御方 DEF_DICE +1；假身在此生成持续 +1 回合。

视觉定位：

```text
防御 / 隐蔽地形
自然遮蔽
```

建议表现：

- 树荫。
- 残墙阴影。
- 岩体投影。
- 边缘柔和的暗绿覆盖。

颜色方向：

```text
深草绿
墨蓝灰低透明度
暖灰阴影
```

要求：

- 掩影不能像黑雾。
- 掩影不能像障碍。
- 掩影不阻挡直线视野的规则认知。
- 掩影可以由完整地图的树冠阴影承担视觉，但规则范围由程序标记。

禁止：

- 画成实体障碍。
- 画成高墙。
- 画成纯黑地块。
- 让玩家误以为阻挡移动或视线。

---

## dusk_rift 黄昏裂隙

规则：青丘单位经过时收集1点狐火。

视觉定位：

```text
夕子聚集形成的黄昏伤口
狐火来源点
危险但可进入
```

黄昏裂隙视觉以专项文档为准：

```text
视觉设计/地图规范/黄昏裂隙视觉规范.md
```

建议表现：

- 地表微光裂纹。
- 夕照红、琥珀橙、暮紫微光。
- 裂纹深处暗红或暮紫阴影。
- 少量狐火余烬，但不形成火焰团。
- 裂隙可以跨越地表，但不可抬高成障碍。

禁止：

- 狐火青裂缝。
- 蓝色空间门。
- 绿色鬼火裂缝。
- 科幻能量裂缝。
- 深坑障碍。
- 大裂谷阻路。

---

## obstacle 障碍

规则：不可进入；阻挡移动和直线视野。

视觉定位：

```text
最明确的阻挡地形
```

建议表现：

- 巨石。
- 古树。
- 残墙。
- 封魔柱。
- 路障。

视觉关键词：

```text
实体
不可进入
阻挡
高轮廓
```

要求：

- 轮廓必须高于普通地形。
- 占格边界明确。
- 必须明显区别于掩影。
- 与地图光源、材质、色温一致。

禁止：

- 做得太低，像普通装饰。
- 只用阴影表现。
- 与掩影视觉混淆。
- 风格比地图主体更写实或更卡通。

# 五、据点地形视觉

## central_objective 中央据点

规则：据点范围为中心格及相邻1圈格；归属方在回合结算时 +2 气运。

视觉定位：

```text
地图核心目标
青丘谷古路节点
战术核心
```

建议表现：

- 低矮圆形石台。
- 旧路标或旗帜。
- 地面狐纹或路线纹。
- 周边石板与道路自然连接。
- 中心格略高于普通地表，但不能像大型神殿。

表现层级：

```text
地表底图：古路与石板连接
据点资源：中心石台 / 旗帜 / 旧路标
程序层：归属色 / 占领范围提示
```

颜色方向：

```text
中立：旧石灰 + 暮金
青丘归属：夕照红 + 琥珀橙 + 暮金
天门归属：执法白金 + 冷灰蓝
```

禁止：

- 把青丘 / 天门归属直接画死在基础图。
- 中央据点做成大建筑遮挡周围格。
- 用满屏光效表现据点。
- 华丽程度远超地图整体，导致突兀。

---

## edge_objective 边缘据点

规则：据点范围为中心格；归属方在回合结算时 +1 气运。

视觉定位：

```text
小型目标点
边境路标
低于中央据点
高于普通格
```

建议表现：

- 小型石基。
- 旗帜。
- 旧路标。
- 地面小型符纹。
- 只强调当前格，不扩散到相邻格。
- 体积低矮，避免误判为障碍。

与中央据点区别：

```text
中央据点：古路节点 + 圆形石台 + 范围感
边缘据点：单格路标 + 小旗帜 + 低矮石基
```

---

## 据点归属视觉

据点拥有归属状态。

归属必须由程序层或覆盖层表现。

| 归属 | 视觉 |
|---|---|
| 无归属 | 旧石灰、低亮暮金纹 |
| 青丘 | 夕照红、琥珀橙、暮金呼吸光 |
| 天门 | 执法白金、冷灰蓝稳定光 |
| 争夺中 | 低频闪烁边线，不改变底图 |

禁止：

```text
在地块底图中画死归属
```

原因：

```text
归属会变化，底图不应承担状态表达
```

# 六、地块状态视觉

地块状态不是基础地形，只是附着在地图格子上的持续效果。

状态必须做成：

```text
程序叠加层 / 透明覆盖层
```

不得画死在地块底图里。

---

## 狐火残留

规则：青丘单位进入带有狐火残留状态的地块：收集1点狐火；敌方单位进入获得迷踪；不阻挡移动和视线；不能生成在障碍格。

视觉定位：

```text
青丘留下的可交互地块状态
夕子经狐族血脉后的可见残痕
```

建议表现：

- 地面上方低透明狐火余烬。
- 小范围夕照色粒子。
- 沿路径流动的暮金微光。
- 不改变地形原本形状。

颜色：

```text
夕照红
琥珀橙
暮金
少量暮紫
```

要求：

- 可见，但不能盖住地形类型。
- 不能盖住棋子底座。
- 不能像黄昏裂隙本体。
- 不能像普通火焰团。

禁止：

- 狐火青粒子。
- 蓝色灵火。
- 绿色鬼火。
- 爆炎火球。

---

## 勘验区

规则：指定1格及相邻格获得勘验区状态；持续到回合结束；带有勘验区状态的狐火残留、黄昏裂隙、掩影效果失效；不阻挡移动和视线。

视觉定位：

```text
天门 / 调查 / 压制性规则区域
```

建议表现：

- 几何扫描纹。
- 细线六边形边框。
- 执法白金色低透明覆盖。
- 可叠加在狐火残留、黄昏裂隙、掩影上，表示其效果被压制但没有移除。

颜色：

```text
执法白金 #D8D2BF
冷灰蓝
墨蓝灰 #2A313B
```

表现要求：

```text
状态压制感
规则清晰
不能像障碍
不能像占领范围
```

# 七、程序提示层

程序提示层不属于地块本体。

包括：

- 六边形网格线。
- 选中框。
- 移动范围。
- 攻击范围。
- 占领范围。
- 据点归属色。
- 勘验区覆盖层。
- 狐火残留覆盖层。

推荐层级：

```text
Layer 1：完整地表底图 / 无缝纹理
Layer 2：特殊地形覆盖
Layer 3：据点 / 障碍 / 装饰
Layer 4：地块状态
Layer 5：程序高亮
Layer 6：棋子
Layer 7：技能预览 / 攻击范围
```

# 八、精灵图生产规范

地块精灵图用于生产和拆分资源，不是游戏最终整图。

精灵图应按以下分类排列：

```text
地表纹理
特殊地形覆盖
据点资源
地块状态
占领范围提示
障碍物
装饰物
```

要求：

- 绿幕背景 `#00FF00` 或透明背景。
- 无 UI。
- 无角色。
- 无宣传特效。
- 文字标签仅允许出现在参考图，不允许进入最终资源切片。
- 地块之间留足切割间距。
- 每个资源保持统一正交45°视角。
- 统一左上光源。
- 统一明亮高对比风格。

# 十、禁止事项

- 新增地形规则。
- 新增地形名称作为规则本体。
- 用视觉表现覆盖规则设定。
- 把地块状态画死进基础地形。
- 把据点归属画死进基础地形。
- 让掩影视觉像障碍。
- 让黄昏裂隙视觉像不可进入深坑。
- 让高台视觉像不可进入山体。
- 让装饰物遮挡路线判断。
- 使用透视投影导致远处地块缩小。
- 将大量独立六边形插画直接拼成最终地图。
- 将狐火、黄昏裂隙默认画成青蓝或绿色鬼火。
