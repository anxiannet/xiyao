# 《夕妖》MVP设定表字段说明

## 文档用途

本文档定义《夕妖》MVP 在 Supabase 中的 `cfg_` 静态设定表。

设定表只保存当前自由对战与 AI 对战需要读取的静态数据。

规则判断由规则引擎实现。

本文档不定义 `match_` 对局运行表、`sim_` 模拟测试表、用户表或资源文件表。

---

# 1. 设计原则

## 1.1 字段只存数据

字段保存稳定、可复用、可查询的数据。

不要把具体规则逻辑写成字段名。

例如不要设计：

```text
support_success_near_dusk_rift
suppresses_foxfire_residue
grants_foxfire_on_enter
```

这些属于规则引擎逻辑。

## 1.2 字段保持通用

字段不能绑定某个具体小队、英雄或机制。

例如地形表不能出现只服务青丘的字段，状态表不能出现只服务某个状态的字段。

## 1.3 好坏不写入状态定义

状态对不同小队、目标和局面可能有不同价值。

例如同一个地图效果可能对青丘有利，对天门不利。

因此状态定义表不使用 `is_negative`。

状态是否有利，应由规则引擎结合状态持有者、观察方、当前阵营、当前目标和当前局面动态判断。

## 1.4 律令与状态分离

律令是玄照可选择颁布的技能或法术定义。

律令颁布后，会在对局中生成对应的全局规则状态。

例如：

```text
禁行令
↓
禁行状态
```

因此：

- `cfg_decrees` 保存可选择的律令定义。
- `cfg_statuses` 保存律令生效后产生的全局状态定义。

## 1.5 字段值高度重复时应移除

如果一个字段在大部分记录中取值都一样，说明它更可能是规则常量，而不是数据。

这类内容应放在规则文档或规则引擎中。

## 1.6 不预留字段

当前 MVP 不需要的字段不进入表。

## 1.7 不用万能字段承载规则

设定表不使用 `metadata`、`effect_json`、`rule_json` 这类万能字段承载核心规则。

规则文本可用 `rule_text` 保存，供阅读、UI 展示与审计。

---

# 2. 通用约定

## 2.1 主键

所有设定表统一使用：

```text
id text primary key
```

`id` 使用稳定可读 key，便于种子数据、日志、调试和 AI 测试。

示例：

```text
qingqiu_envoys
suling
fox_mark
central_objective
test_map_a
```

## 2.2 显示名称

所有需要展示名称的表统一使用：

```text
name text not null
```

## 2.3 规则文本

需要向玩家或开发者解释规则的表，使用：

```text
rule_text text not null
```

`rule_text` 只用于展示与审计，不作为规则引擎的唯一执行来源。

# 3. 当前设定表总览

| 表名 | 用途 |
|---|---|
| cfg_squads | 小队 |
| cfg_units | 英雄单位 |
| cfg_skills | 核心技能 |
| cfg_statuses | 状态与持续效果 |
| cfg_terrains | 地形 |
| cfg_decrees | 律令 |
| cfg_maps | 地图模板 |
| cfg_map_tiles | 地图格子 |

---

# 4. cfg_squads 小队表

## 用途

定义可选小队。

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 小队ID |
| name | text | 是 | 小队名称 |
| faction | text | 是 | 所属势力 |
| playstyle | text | 是 | 小队玩法定位 |

## 关系

```text
cfg_squads 1 对多 cfg_units
```

---

# 5. cfg_units 英雄单位表

## 用途

定义英雄单位的基础数据。

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 单位ID |
| squad_id | text | 是 | 所属小队，引用 `cfg_squads.id` |
| name | text | 是 | 单位名称 |
| is_leader | boolean | 是 | 是否队长 |
| hp | integer | 是 | 最大生命值 |
| mv | integer | 是 | 移动值 |
| atk_dice | integer | 是 | 基础攻击骰数量 |
| def_dice | integer | 是 | 基础防御骰数量 |
| dmg | integer | 是 | 基础伤害 |
| rng | integer | 是 | 普通攻击距离 |

## 关系

```text
cfg_units 多 对 1 cfg_squads
cfg_units 1 对多 cfg_skills
```

说明：

玄照可以拥有多个律令类技能记录。

禁行令、戒严令、追捕令是玄照可选择使用的技能或法术。

它们颁布后产生对应的全局规则状态。

---

# 6. cfg_skills 核心技能表

## 用途

定义英雄技能。

当前 MVP 中，多数英雄只有一个核心技能。

玄照可以有多个律令类技能记录。

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 技能ID |
| unit_id | text | 是 | 所属单位，引用 `cfg_units.id` |
| name | text | 是 | 技能名称 |
| rule_text | text | 是 | 技能规则文本 |

## 关系

```text
cfg_skills 多 对 1 cfg_units
```

---

# 7. cfg_statuses 状态与持续效果表

## 用途

定义会在对局中持续存在、需要显示或记录的效果。

状态不只包括单位状态，也包括地形状态、区域状态、队伍状态与全局规则状态。

例如：

- 狐印：单位状态
- 迷踪：单位状态
- 逃逸：单位状态
- 狐火充盈：小队状态
- 狐火残留：地形状态
- 勘验区：区域状态
- 禁行状态：全局规则状态
- 戒严状态：全局规则状态
- 追捕状态：全局规则状态

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 状态ID |
| name | text | 是 | 状态名称 |
| status_scope | text | 是 | 状态作用域：`unit` / `tile` / `zone` / `team` / `global` |
| rule_text | text | 是 | 状态规则文本 |

## 说明

状态定义不记录好坏。

状态对谁有利、对谁有害，由规则引擎在具体局面中判断。

---

# 8. cfg_terrains 地形表

## 用途

定义地图格子的基础地形。

据点也是地形。

中央据点、边缘据点与普通格、高台、掩影、黄昏裂隙、障碍都放在本表。

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 地形ID |
| name | text | 是 | 地形名称 |
| blocks_movement | boolean | 是 | 是否阻挡移动 |
| blocks_line_of_sight | boolean | 是 | 是否阻挡直线视野 |
| can_place_unit | boolean | 是 | 是否允许单位停留 |
| rule_text | text | 是 | 地形规则文本 |

---

# 9. cfg_decrees 律令表

## 用途

定义玄照可选择颁布的律令。

律令属于技能或法术定义。

律令颁布后，会在对局中产生对应的全局规则状态。

例如：

```text
禁行令
↓
禁行状态
```

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 律令ID |
| name | text | 是 | 律令名称 |
| status_id | text | 是 | 颁布后产生的全局状态，引用 `cfg_statuses.id` |
| rule_text | text | 是 | 律令规则文本 |

## 关系

```text
cfg_decrees 多 对 1 cfg_statuses
```

---

# 10. cfg_maps 地图模板表

## 用途

定义自由对战和 AI 对战使用的地图模板。

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 地图ID |
| name | text | 是 | 地图名称 |

## 关系

```text
cfg_maps 1 对多 cfg_map_tiles
```

---

# 11. cfg_map_tiles 地图格子表

## 用途

定义地图模板中的六角格。

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 格子ID |
| map_id | text | 是 | 所属地图，引用 `cfg_maps.id` |
| q | integer | 是 | hex axial 坐标 q |
| r | integer | 是 | hex axial 坐标 r |
| terrain_id | text | 是 | 地形，引用 `cfg_terrains.id` |
| deployment_owner | text | 否 | 部署区归属：`qingqiu` / `tianmen`；非部署区为空 |

## 关系

```text
cfg_map_tiles 多 对 1 cfg_maps
cfg_map_tiles 多 对 1 cfg_terrains
```

## 约束

```sql
unique (map_id, q, r)
```

# 14. RLS建议

```text
select: allow all
insert/update/delete: service_role only
```

---

# 15. 审计结论

当前 `cfg_` 设定表只保存自由对战和 AI 对战需要读取的静态数据。

规则判断不通过字段名表达。

小队专属逻辑不进入字段结构。

据点归入地形。

状态表统一承载单位状态、地形状态、区域状态、小队状态与全局规则状态。

状态不定义绝对好坏。

律令保留为独立设定表。

律令颁布后产生对应的全局规则状态。

具体结算逻辑由规则文档和规则引擎负责。
