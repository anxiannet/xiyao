# 《夕妖》MVP设定表字段说明

## 文档用途

本文档定义《夕妖》MVP 在 Supabase 中的最小 `cfg_` 设定表字段。

本文只覆盖静态设定表。

不包含：

- `match_` 对局运行表
- `sim_` AI模拟表
- `user_` 用户表
- `asset_` 资源表

---

# 1. 本次审计结论

当前 MVP 只做：

- 自由对战
- 玩家 vs AI
- AI vs AI

因此设定表只服务当前《夕妖》MVP。

不在数据库业务表中预留：

- 多游戏系统
- 多规则版本系统
- 战役系统
- 教学关卡系统
- 解锁奖励系统
- 皮肤资源系统
- 商城系统

---

# 2. 删除的非必要设定表

## 2.1 删除 `cfg_games`

原因：

当前数据库只服务《夕妖》MVP。

不需要用业务表记录“这是哪个游戏”。

多游戏平台是未来需求，不进入 MVP。

## 2.2 删除 `cfg_rulesets`

原因：

当前 MVP 只存在一个有效规则版本。

规则版本由：

- Git 提交
- Supabase migration
- 种子数据文件

管理，不需要单独业务表。

## 2.3 删除 `cfg_campaigns`

原因：

单人战役已移出 MVP。

## 2.4 删除 `cfg_scenarios`

原因：

教学关卡与战役关卡已移出 MVP。

# 3. 删除的非必要字段

以下字段不再出现在 MVP 设定表中。

| 字段 | 删除原因 |
|---|---|
| game_id | 当前只服务《夕妖》，不做多游戏 |
| ruleset_id | 当前只做一个 MVP 规则版本 |
| name_en | 当前界面与文档以中文为主，不做英文多语言 |
| metadata | 万能扩展字段，不符合字段单一原则 |
| tags | 当前规则对象有限，不需要标签数组检索 |
| is_enabled | 当前种子数据即有效数据，不做后台启停 |
| sort_order | 当前列表顺序可由主键或固定读取顺序控制 |
| created_at | 设定表由 migration/seed 管理，不作为业务审计字段 |
| updated_at | 设定表由 migration/seed 管理，不作为业务审计字段 |
| cooldown_type | 当前 MVP 技能没有冷却系统 |
| skill_type | 当前每名单位只有一个核心技能，不需要技能类型 |
| effect_json | 规则不放入万能 JSON，拆成明确字段 |
| metadata_json | 万能扩展字段，不进入 MVP |
| unlock_rule_json | 解锁系统不进入 MVP |
| reward_json | 奖励系统不进入 MVP |

# 4. 字段设计原则

## 4.1 字段单一

一个字段只表达一个含义。

例如：

```text
hp
mv
atk_dice
def_dice
dmg
rng
```

不要合并成：

```text
stats_json
```

## 4.2 字段独立

字段之间尽量减少重复计算与嵌套依赖。

例如单位表记录 `squad_id`，不再同时记录 `squad_name`。

## 4.3 不预留

当前 MVP 没有用到的字段不进入表。

例如：

- 冷却
- 英文名
- 皮肤资源
- 解锁条件
- 未来技能位
- 多版本规则

## 4.4 不用万能 JSON 承载核心规则

核心规则应尽量拆成可查询、可约束、可测试的独立字段。

只有当字段确实是列表型固定数据时，才使用数组。

当前 MVP 设定表尽量不使用 `jsonb`。

# 6. 通用约定

## 6.1 主键

所有设定表统一使用：

```text
id text primary key
```

说明：

设定表是人工维护的静态种子数据。

使用稳定可读主键更适合调试、日志与种子数据维护。

例如：

```text
suling
qingqiu_envoys
fox_mark
central_objective
```

## 6.2 外键

外键统一使用对应表的 `id`。

例如：

```text
cfg_units.squad_id -> cfg_squads.id
cfg_skills.unit_id -> cfg_units.id
cfg_map_tiles.map_id -> cfg_maps.id
```

## 6.3 显示名

统一字段：

```text
name text not null
```

当前 MVP 不做多语言。

# 7. cfg_squads 小队表

## 用途

定义当前 MVP 的两支小队。

当前数据：

- 青丘使团
- 天门执法队

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 小队ID，例如 `qingqiu_envoys` |
| name | text | 是 | 小队名称 |
| faction | text | 是 | 所属势力，例如 `qingqiu` / `tianmen` |
| squad_size | integer | 是 | 小队正式英雄数量 |
| playstyle | text | 是 | 小队玩法定位 |

## 关系

```text
cfg_squads 1 对多 cfg_units
```

---

# 8. cfg_units 单位表

## 用途

定义英雄单位的基础数值。

当前 MVP 单位：

- 苏绫、阿照、青萝、琉尾
- 玄照、白烬、赤霄

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 单位ID，例如 `suling` |
| squad_id | text | 是 | 所属小队，引用 `cfg_squads.id` |
| name | text | 是 | 单位名称 |
| is_leader | boolean | 是 | 是否队长 |
| hp | integer | 是 | 最大生命值 |
| mv | integer | 是 | 移动值，正式英雄最低为2 |
| atk_dice | integer | 是 | 基础攻击骰数量 |
| def_dice | integer | 是 | 基础防御骰数量 |
| dmg | integer | 是 | 基础伤害 |
| rng | integer | 是 | 普通攻击距离 |

## 关系

```text
cfg_squads 1 对多 cfg_units
cfg_units 1 对 1 cfg_skills
```

说明：

当前 MVP 每名英雄只有一个核心技能。

因此 `cfg_skills.unit_id` 应设置唯一约束。

---

# 9. cfg_skills 技能表

## 用途

定义每名英雄的唯一核心技能。

当前 MVP 技能：

- 狐步
- 假身
- 狐火引路
- 扰弦
- 颁令
- 勘验
- 护阵

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 技能ID，例如 `fox_step` |
| unit_id | text | 是 | 所属单位，引用 `cfg_units.id`，并唯一 |
| name | text | 是 | 技能名称 |
| ap_cost | integer | 是 | AP 消耗 |
| range_value | integer | 否 | 技能距离；无距离概念时为空 |
| target_type | text | 是 | 目标类型：`enemy_unit` / `ally_unit` / `tile` / `self` / `decree` |
| rule_text | text | 是 | 完整规则文本，供 UI 与规则审计读取 |

## 删除字段说明

不设置冷却字段。

原因：当前 MVP 没有冷却系统。

不设置技能类型字段。

原因：当前每名英雄只有一个核心技能。

不设置效果 JSON。

原因：核心规则暂由规则引擎代码实现，数据库只保存可读规则文本与必要参数。

# 10. cfg_statuses 状态表

## 用途

定义当前 MVP 状态。

当前状态：

- 狐印
- 迷踪
- 逃逸
- 狐火充盈

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | text | 是 | 状态ID，例如 `fox_mark` |
| name | text | 是 | 状态名称 |
| is_negative | boolean | 是 | 是否负面状态 |
| duration_type | text | 是 | 持续类型：`turns` / `until_next_move` / `until_next_attack` / `round_end` |
| duration_value | integer | 否 | 持续数值，例如狐印为1 |
| stack_rule | text | 是 | 叠加规则：`refresh` / `none` |
| remove_timing | text | 是 | 移除时机，例如 `round_end` / `after_trigger` |
| rule_text | text | 是 | 完整规则文本 |

## 当前约束

狐印：

```text
stack_rule = refresh
```

迷踪：

```text
duration_type = until_next_move
remove_timing = after_trigger
```

# 19. cfg_map_tiles 地图格子表

# 20. 索引建议

只为当前查询路径建立索引。

```sql
create index on cfg_units (squad_id);
create unique index on cfg_skills (unit_id);
```

其他索引等真实查询出现后再加。

# 22. 审计结论

- 小队
- 英雄
- 核心技能
- 状态
- 资源

- 覆盖层

- 据点
- 律令

- 地图格子

本版本特意不包含：

- 游戏表
- 规则版本表
- 战役表
- 关卡表
- 用户表
- 资源表
- 未来扩展字段
- 万能 metadata 字段
- 万能 effect_json 字段
