# 《夕妖》MVP设定表字段说明

## 文档用途

本文档定义《夕妖》MVP 在 Supabase 中的 `cfg_` 设定表字段。

本文只覆盖静态设定表。

不包含：

- `match_` 对局运行表
- `sim_` AI模拟表
- `user_` 用户表
- `asset_` 多游戏资源表

当前 MVP 只保留自由对战、玩家 vs AI、AI vs AI。

不包含单人战役、教学关卡、解锁奖励。

---

# 1. Supabase设计原则

## 1.1 数据库类型约定

| 类型 | 用途 |
|---|---|
| uuid | 主键、外键 |
| text | 短文本、枚举值、key |
| integer | 整数数值 |
| boolean | 开关值 |
| text[] | 标签、列表型短文本 |
| jsonb | 规则参数、复杂配置 |
| timestamptz | 创建、更新时间 |

## 1.2 主键规则

所有 `cfg_` 表统一使用：

```text
id uuid primary key default gen_random_uuid()
```

## 1.3 多游戏与多版本规则

所有核心设定表都应带：

```text
game_id uuid
ruleset_id uuid
```

用途：

- `game_id` 区分不同游戏。
- `ruleset_id` 区分不同规则版本。

## 1.4 可读key规则

除 `id` 外，每张设定表都应有一个可读 `key`。

例如：

```text
qingqiu_envoys
suling
fox_mark
central_objective
```

程序内部可以用 `id` 做关联。

策划、种子数据、调试日志可以用 `key` 阅读。

## 1.5 通用字段

除特殊说明外，所有 `cfg_` 表建议包含以下通用字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid | 主键 |
| game_id | uuid | 所属游戏，引用 `cfg_games.id` |
| ruleset_id | uuid | 所属规则版本，引用 `cfg_rulesets.id` |
| key | text | 稳定可读key，同一 ruleset 内唯一 |
| name_zh | text | 中文显示名 |
| name_en | text | 英文或拼音显示名，可为空 |
| description | text | 说明文本 |
| tags | text[] | 标签数组 |
| sort_order | integer | 显示与读取顺序 |
| is_enabled | boolean | 是否启用 |
| created_at | timestamptz | 创建时间 |
| updated_at | timestamptz | 更新时间 |

建议唯一约束：

```sql
unique (ruleset_id, key)
```

部分全局表如 `cfg_games` 不需要 `game_id` 与 `ruleset_id`。

# 2. cfg_games 游戏表

## 用途

定义一个游戏项目。

当前 MVP 只需要一条：

```text
xi_yiao
```

## 字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | uuid | 是 | 游戏ID |
| key | text | 是 | 游戏key，例如 `xi_yiao` |
| name_zh | text | 是 | 中文名，例如《夕妖》 |
| name_en | text | 否 | 英文名或拼音名 |
| description | text | 否 | 游戏说明 |
| status | text | 是 | 状态：draft / active / archived |
| default_ruleset_id | uuid | 否 | 默认规则版本，引用 `cfg_rulesets.id` |
| metadata | jsonb | 否 | 扩展信息 |
| created_at | timestamptz | 是 | 创建时间 |
| updated_at | timestamptz | 是 | 更新时间 |

## 关系

```text
cfg_games
1 对多
cfg_rulesets
```

# 18. RLS建议

设定表通常是公开只读数据。

MVP 阶段建议：

- 前端匿名用户可以读取已启用的 `cfg_` 数据。
- 只有服务端或管理员可以写入。

建议策略：

```text
select: is_enabled = true
insert/update/delete: service_role only
```
