# 《夕妖》核心标签系统 V1

## 目标

本文档定义《夕妖》MVP 与后续扩展战帮共用的底层标签。

设计目标：

- 技能和宝物优先识别标签，而不是识别具体战帮名或具体技能名。
- MVP 只实现必要标签。
- 后续新增战帮时，优先给新状态、覆盖层、召唤物配置标签，而不是修改旧技能。

---

# 1. 标签设计原则

## 1.1 规则文本优先写机制

推荐写法：

```text
清除 1 个敌方召唤物。
```

不推荐写法：

```text
清除 1 个假身。
```

原因：

- 假身是青丘的具体召唤物。
- 召唤物是可扩展机制。
- 未来机关兽、影卫、纸人都可以复用同一套规则。

## 1.2 示例可以写具体名词

规则主体写通用机制。

示例可以写当前 MVP 名词。

例如：

```text
清除 1 个敌方召唤物。
MVP 示例：假身属于敌方召唤物。
```

# 2. 单位标签 UnitTag

| 标签 | 含义 | MVP 示例 | 后续扩展示例 |
|---|---|---|---|
| unit | 正式单位 | 苏绫、玄照、白烬 | 所有正式角色 |
| leader | 队长单位 | 苏绫、玄照 | 各战帮队长 |
| summon | 召唤物 | 假身 | 机关兽、影卫、纸人 |
| support_source | 可提供支援 | 正式单位、假身 | 战旗、影卫 |

## 2.1 正式单位 unit

正式单位可以：

- 激活。
- 移动。
- 攻击。
- 使用技能。
- 占领据点。
- 被击败后给敌方气运。

## 2.2 队长 leader

队长是战帮核心机制来源。

MVP 原则：

- 青丘队长苏绫是狐印的唯一稳定来源。
- 天门队长玄照是追缉的唯一稳定来源。

队长阵亡后，对应战帮的核心标记生产能力明显下降。

## 2.3 召唤物 summon

召唤物默认规则：

- 占格。
- 阻挡移动。
- 不阻挡直线视野，除非召唤物规则特别说明。
- 不占领据点。
- 不主动攻击，除非规则特别说明。
- 承受 1 次攻击或技能后消失，除非规则特别说明。

MVP 示例：

- 假身。

# 3. 状态标签 StatusTag

| 标签 | 含义 | MVP 示例 | 后续扩展示例 |
|---|---|---|---|
| buff | 正面状态 | 护盾 | 加速、攻击强化 |
| debuff | 负面状态 | 狐印、追缉、封印、迷踪、易伤 | 中毒、灼烧、梦魇 |
| mark | 标记状态 | 狐印、追缉 | 猎印、血印、梦印 |
| control | 控制状态 | 封印 | 沉默、眩晕、冻结 |
| defense | 防御状态 | 护盾 | 格挡、圣佑 |

## 3.1 正面状态 buff

默认只能施加给友方单位。

若可施加给敌方，必须在规则中明确写出。

## 3.2 负面状态 debuff

默认只能施加给敌方单位。

若可施加给友方，必须在规则中明确写出。

## 3.3 标记 mark

标记是一种常见负面状态。

标记通常用于：

- 触发额外攻击收益。
- 提供支援。
- 作为宝物或技能的前置条件。

MVP 示例：

- 狐印：青丘攻击带狐印目标时，支援面自动成功。
- 追缉：天门攻击带追缉目标时，降低其防御骰。

## 3.4 控制 control

控制是更强的负面状态。

MVP 示例：

- 封印：目标无法使用主动技能。

封印本身不是“不可净化”。

当前 MVP 的白烬只能进行友方净化，因此不能净化敌方身上的封印。

后续若出现可选择敌方单位的净化、驱散或解除控制能力，可以移除封印。

# 4. 覆盖层标签 ZoneTag

| 标签 | 含义 | MVP 示例 | 后续扩展示例 |
|---|---|---|---|
| beneficial_zone | 对生成方有利的覆盖层 | 律令领域 | 圣光领域、机关阵地 |
| harmful_zone | 对敌方有害的覆盖层 | 狐火残留对天门而言 | 毒雾、梦魇领域 |
| resource_zone | 可产生资源的覆盖层 | 狐火残留 | 灵泉、矿脉 |
| control_zone | 影响移动或站位的覆盖层 | 律令领域 | 重力阵、禁行阵 |

## 4.1 覆盖层互斥

同一格可存在基础地形和覆盖层。

但敌对覆盖层互斥。

MVP：

```text
狐火残留 ↔ 律令领域
```

后生成者覆盖前者。

## 4.2 有害覆盖层 harmful_zone

“有害”是相对阵营概念。

例如：

- 狐火残留对青丘是资源覆盖层。
- 狐火残留对天门是有害覆盖层。
- 律令领域对天门是有益覆盖层。
- 律令领域对青丘是有害控制覆盖层。

# 5. 资源标签 ResourceTag

| 标签 | 含义 | MVP 示例 |
|---|---|---|
| warband_resource | 战帮资源 | 狐火、威仪 |
| victory_resource | 胜利资源 | 气运 |

## 5.1 战帮资源

战帮资源服务于战帮机制。

MVP 示例：

- 青丘：狐火。
- 天门：威仪。

## 5.2 胜利资源

气运是胜利资源。

不属于任何单一战帮。

# 7. 后续扩展规范

新增战帮时，必须先定义：

1. 队长专属机制。
2. 战帮资源。
3. 至少 1 个核心状态或覆盖层。
4. 这些对象对应的标签。
5. 与现有通用规则的交互。

新增技能或宝物时，应优先引用：

- 负面状态。
- 标记状态。
- 控制状态。
- 有害覆盖层。
- 召唤物。
- 战帮资源。

尽量避免引用：

- 狐印。
- 追缉。
- 假身。
- 狐火残留。

除非该技能或宝物明确属于对应战帮主题。
