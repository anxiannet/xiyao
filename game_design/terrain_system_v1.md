# 《夕妖》地形系统 V1.1

## 核心结论

《夕妖》的地图不依赖固定手绘棋盘。

MVP 采用：

- 背景图负责氛围
- 六角网格由程序生成
- 地形标签由程序摆放
- 地形效果由规则系统控制
- 特殊地形加载覆盖美术资源
- 地形残留由程序动态生成和清除

地图背景不是规则本体。

规则读取的是格子标签、状态覆盖层和持续时间。

---

# 地图分层

每张地图由四层构成。

## 1. 背景层 background_layer

纯视觉氛围。

例如：

- 天门关夜色
- 青丘谷黄昏
- 暮渊裂谷余烬
- 室内神殿
- 林中祭坛

背景不参与规则计算。

## 2. 网格层 grid_layer

程序生成六角格。

每个格子拥有坐标。

推荐使用 axial 坐标：

```text
q, r
```

## 3. 地形标签层 terrain_layer

程序给格子添加地形标签。

例如：

```text
terrain: cover_shadow
terrain: high_ground
terrain: dusk_rift
terrain: daylight_altar
```

## 4. 覆盖资源层 overlay_layer

特殊地形、临时地形、残留效果需要加载覆盖美术资源。

例如：

```text
overlay: foxfire_residue
asset: overlays/foxfire_residue_01.png
turns_remaining: 1
```

覆盖层参与规则计算。

---

# 地形与覆盖层区别

## 地形 terrain

地图生成时存在。

通常持续整局。

例如：

- 高台
- 掩影
- 昼曜令台
- 中央据点

## 覆盖 overlay

由技能、宝物、事件生成。

通常有持续时间。

例如：

- 狐火残留
- 临时障碍
- 圣光结界
- 关闭的裂隙
- 夕门残影

# 名词修正：暮炁与狐火

原“夕火”容易与青丘技能“狐火”混淆。

从 V1.1 起：

世界本源与资源改名为：

## 暮炁

含义：黄昏本源、变化之力、暮君残留力量。

青丘技能状态仍叫：

## 狐火

含义：青丘狐族用暮炁点燃的派生火痕，不是世界本源。

区分：

- 暮炁：资源 / 本源 / 天象系统。
- 狐火：状态 / 地格残留 / 青丘战术标记。

# 地形变体原则

同一类规则地形，在不同场景中可以有不同美术表现。

例如：

## 掩影类

室外表现：林地、竹林、狐影林。

室内表现：阴影、帷幕、屏风、暗廊。

规则统一为：

```text
cover_shadow
```

## 高台类

室外表现：山坡、石台、城墙。

室内表现：楼阁、祭台、二层回廊。

规则统一为：

```text
high_ground
```

## 裂隙类

室外表现：黄昏裂隙、暮炁裂缝。

室内表现：破碎镜门、暮门、裂纹阵眼。

规则统一为：

```text
dusk_rift
```

# MVP地形总表

## 1. 普通格 plain

基础地形。

无特殊效果。

用途：保证地图可读性。

技能关系：无。

宝物关系：可被山河印临时改为障碍。

---

## 2. 中央据点 central_objective

规则：

站上后可执行占领。

回合结束时控制者获得气运。

天门关系：

- 赤霄护阵在中央据点上占点权重+1。
- 镇守囊强化中央据点控制。

青丘关系：

- 琉尾扰弦可降低据点上敌人的占点权重。
- 青丘通常不适合硬抢中央，需要通过边缘点牵制。

宝物关系：

- 天门钥：立即占领相邻中央据点。
- 镇地印：指定据点，敌方占点权重-1。

美术覆盖：

据点需要单独加载地面标识资源。

```text
assets/terrain/objective_central.png
```

---

## 3. 边缘据点 edge_objective

规则：

地图边缘的占点目标。

回合结束时控制者获得气运。

天门关系：

- 天门移动慢，难以快速响应边缘据点。
- 律令旗不能额外强化边缘点。

青丘关系：

- 苏绫队长光环使青丘从边缘格移动+1。
- 青丘迷踪囊主要围绕边缘据点运作。

宝物关系：

- 灵狐羽从边缘格出发时更强。
- 镇地印可压制青丘偷点。

美术覆盖：

```text
assets/terrain/objective_edge.png
```

---

## 4. 高台 high_ground

美术变体：

- 山坡
- 城墙
- 石台
- 楼阁
- 二层回廊

规则：

远程攻击距离+1。

若技能特别指定高台狙击，可改为攻击骰+1。

两者不叠加。

天门关系：

- 玄照在高台使用律令封禁时距离+1。
- 高台可以辅助识破假身。

青丘关系：

- 琉尾在高台攻击时可击退目标1格。
- 青丘可利用高台骚扰中央据点。

宝物关系：

- 定风珠可免疫高台击退。
- 山河印可封堵通往高台的路径。

美术覆盖：

高台必须加载高度标识，避免玩家误判。

```text
assets/terrain/high_ground_marker.png
```

---

## 5. 掩影 cover_shadow

美术变体：

- 林地
- 竹林
- 狐影林
- 阴影
- 屏风
- 帷幕
- 暗廊

规则：

防御方位于掩影时，防御骰+1。

假身在掩影格生成时持续+1回合。

天门关系：

- 天门不喜欢掩影地形，因为封印容易被假身诱导。
- 昼曜令台可清除附近掩影对假身的增益。

青丘关系：

- 阿照在掩影生成假身更强。
- 影遁符可移动到相邻掩影格。

宝物关系：

- 狐影铃在掩影中生成的假身持续+1回合。
- 圣光结界可清除进入范围的假身。

美术覆盖：

室外：

```text
assets/terrain/forest_shadow.png
```

室内：

```text
assets/terrain/indoor_shadow.png
```

---

## 6. 黄昏裂隙 dusk_rift

美术变体：

- 黄昏裂缝
- 暮炁裂隙
- 破碎镜门
- 暮门
- 裂纹阵眼

规则：

青丘单位可通过裂隙进行特殊移动。

单位经过裂隙时增加暮炁值。

部分裂隙可成对连接。

天门关系：

- 石门令可关闭裂隙1回合。
- 山河印可封堵裂隙入口。

青丘关系：

- 苏绫从裂隙移动后攻击可施加狐火。
- 暮灯在裂隙旁使用额外提升暮炁。
- 裂隙针可临时打开裂隙。

宝物关系：

- 石门令关闭。
- 裂隙针打开。
- 暮灯强化。

美术覆盖：

常驻裂隙：

```text
assets/terrain/dusk_rift.png
```

关闭状态：

```text
assets/overlays/rift_sealed.png
```

临时裂隙：

```text
assets/overlays/rift_temporary.png
```

---

## 7. 昼曜令台 daylight_altar

美术变体：

- 天门令台
- 神像台
- 昼曜阵眼
- 审判石座
- 室内法坛

规则：

站在令台上的天门单位封印效果增强。

令台可清除附近狐火残留。

可识破附近假身。

天门关系：

- 玄照封印范围+1。
- 白烬净化可额外清除狐火。

青丘关系：

- 青丘应避免在令台附近堆假身或狐火残留。
- 青丘可通过边缘据点逼天门离开令台。

宝物关系：

- 昼曜令在令台上使用时额外清除狐火。
- 狐影囊在令台附近效果降低。

美术覆盖：

```text
assets/terrain/daylight_altar.png
```

---

## 8. 圣坛 sanctuary

美术变体：

- 晨辉圣坛
- 医疗阵台
- 祈愿树
- 灵泉
- 室内香案

规则：

站上后可进行回复或净化互动。

连续使用可能触发净化代价。

天门关系：

- 白烬在圣坛治疗增强。
- 圣辉囊在圣坛上收益更高。

青丘关系：

- 青丘可利用圣坛恢复脆弱单位。
- 但连续使用会移除自身增益或狐火。

宝物关系：

- 晨辉盏在圣坛使用额外回复1 HP。
- 净世符在圣坛附近可影响相邻单位。

美术覆盖：

```text
assets/terrain/sanctuary.png
```

---

## 9. 窄路 choke_point

美术变体：

- 城门
- 桥
- 石缝
- 回廊
- 山道

规则：

单位通过时路径受限。

不能被多个单位同时穿越。

天门关系：

- 赤霄在窄路护阵时占点权重+1。
- 天门适合用窄路逼迫正面战。

青丘关系：

- 青丘不适合在窄路硬拼。
- 可用假身堵路或引诱封印。

宝物关系：

- 山河印可临时制造窄路/障碍。
- 灵狐羽可帮助绕过窄路。

美术覆盖：

```text
assets/terrain/choke_point_marker.png
```

---

## 10. 障碍 obstacle

美术变体：

- 岩石
- 倒塌柱子
- 大树
- 屏风
- 碎墙

规则：

不可进入。

阻挡直线视野。

天门关系：

- 山河印可生成临时障碍。
- 天门可用障碍封锁裂隙入口。

青丘关系：

- 青丘可绕障碍移动。
- 阿照可利用障碍旁生成假身制造误判。

美术覆盖：

常驻障碍：

```text
assets/terrain/obstacle.png
```

临时障碍：

```text
assets/overlays/temporary_obstacle.png
```

# 狐火残留 foxfire_residue

狐火残留不是基础地形。

它是青丘技能或宝物生成的覆盖层。

## 生成来源

- 青萝：狐火引路。
- 断影香：清除假身后留下狐火残留。
- 狐火珠：指定格生成狐火残留。
- 苏绫从裂隙或掩影攻击后，可在目标格留下狐火残留。

## 规则效果

狐火残留持续1回合。

若位于边缘格、掩影格或黄昏裂隙旁，持续+1回合。

青丘单位经过狐火残留：

- 获得1点暮炁。

天门单位经过狐火残留：

- 获得迷踪。

天门单位站在狐火残留上使用封印：

- 不受额外惩罚，但狐火残留不会自动消失。

## 反制方式

- 白烬晨辉净化可清除狐火残留。
- 昼曜令台清除附近狐火残留。
- 净世符可清除目标格狐火残留。
- 昼曜令可在令台上额外清除狐火。

## 美术资源

狐火残留必须有独立覆盖资源。

```text
assets/overlays/foxfire_residue_01.png
assets/overlays/foxfire_residue_02.png
assets/overlays/foxfire_residue_indoor.png
```

室外表现：赤金狐火、地面火痕。

室内表现：墙影上的狐火纹、地砖上的金红裂纹。

# 程序生成规则

## 地图模板

程序使用模板生成：

```text
map_size: small | medium | large
background_theme: tianmen | qingqiu | muyuan | indoor | forest
terrain_budget:
  high_ground: 2
  cover_shadow: 4
  dusk_rift: 2
  daylight_altar: 1
  central_objective: 1
  edge_objective: 2
asset_theme:
  outdoor | indoor | ruins | forest | temple
```

## 覆盖资源加载

每个格子可拥有：

```text
base_terrain
terrain_asset
overlay_list
status_tags
turns_remaining
```

示例：

```json
{
  "q": 1,
  "r": -2,
  "terrain": "cover_shadow",
  "terrain_asset": "assets/terrain/forest_shadow.png",
  "overlays": [
    {
      "type": "foxfire_residue",
      "asset": "assets/overlays/foxfire_residue_01.png",
      "turns_remaining": 2
    }
  ]
}
```

# 地形与AI

AI需要理解：

- 天门优先占昼曜令台、中央据点、高台。
- 青丘优先利用掩影、边缘据点、黄昏裂隙。
- 赤霄应优先进入窄路与中央据点。
- 阿照应优先在掩影生成假身。
- 青萝应优先在边缘、裂隙、掩影附近留下狐火残留。
- 苏绫应优先从裂隙或边缘格发动攻击。
- 白烬应优先清理狐火残留、迷踪和易伤。

# 结论

《夕妖》的地图本质是规则网格，不是固定美术图。

背景提供氛围。

网格提供坐标。

地形标签提供玩法。

覆盖层提供动态变化。

狐火残留、临时裂隙、封锁裂隙等效果必须由程序生成，并加载对应覆盖美术资源。
