# 《夕妖》MVP机制审计总表

## 用途

本文档用于集中审查当前 MVP 机制是否闭环、是否程序友好、是否 AI 友好，以及是否存在职责重叠、规则残留或实现风险。

当前 MVP 只审计 `game_design/README.md` 列入的有效规则。

归档文档与未来扩展不作为当前实现依据。

---

# 0. 总体结论

| 审计项 | 结论 |
|---|---|
| 是否具备完整可玩闭环 | 是。选择小队、生成地图、部署、轮流激活、掷骰、明确占领、结算气运均已成立 |
| 是否具备两支小队差异 | 是。青丘偏路线、狐火、假身与充盈爆发；天门偏律令时机、勘验、护阵与阵地压制 |
| 是否程序友好 | 是。机制可由标签、状态、区域、触发器、据点归属和数值字段表达 |
| 是否 AI 友好 | 中高。青丘狐火自动充盈，天门只需选择颁令时机、律令类型与勘验区位置 |
| 当前最大风险 | AI 需要理解“占领动作”和“据点归属保持”，不能只按站位即时评分 |
| 当前建议 | 可以进入 MVP 实现；下一步补数据结构字段表与 AI 行为评分表 |

# 1. A级问题处理结果

| 原问题 | 处理结果 | 结论 |
|---|---|---|
| 封印当前无稳定来源 | 已从当前 MVP 规则、标签、命名、战役与审计中移除 | 已解决 |
| 律令默认大回合开始生效，队长时机感弱 | 改为玄照主动技能“颁令”，1AP，每大回合最多1次 | 已解决 |
| 追捕令读取“本回合移动过”会追溯历史 | 改为追捕令生效后，敌方移动才获得逃逸 | 已解决 |
| 律令是否追溯过去行为 | 明确不追溯，只影响颁布后的事件与状态 | 已解决 |
| 据点控制权算法 | 已改为据点归属制：执行占领动作后才改变归属，离开不改变归属 | 已解决 |
| 据点范围 | 中央据点为中心格+相邻1圈；边缘据点为中心格 | 已解决 |
| 狐印持续刷新 | 重复施加刷新为1回合，不叠加层数 | 已解决 |
| 迷踪最低移动值 | 下一次移动距离-1，最低保留1格移动能力；正式英雄 MV 最低为2 | 已解决 |
| 琉尾占点权重 -1 持续 | 已直接移除，不再影响据点算法 | 已解决 |
| 护阵触发次数 | 护阵目标第一次受到攻击时由赤霄承受，触发后护阵结束 | 已解决 |

# 4. 回合与行动闭环审计

| 项目 | 当前规则 | 闭环 | 程序友好 | AI友好 | 风险 |
|---|---|---|---|---|---|
| 大回合 | 一局 3 个大回合 | 是 | 高 | 高 | 无明显风险 |
| 激活 | 双方轮流激活，每单位每大回合 1 次 | 是 | 高 | 高 | 需防止重复激活 |
| AP | 每名单位每次激活 2 AP | 是 | 高 | 高 | 核心技能 AP 消耗需统一字段 |
| 动作 | 移动、攻击、核心技能、占领、待命 | 是 | 高 | 高 | AI 需动作评分 |
| 颁令时机 | 玄照激活时消耗 1AP 主动颁令 | 是 | 高 | 中 | AI 需要判断早颁、晚颁或不颁 |
| 占领动作 | 1AP，满足据点范围内正式单位数量优势后改变据点归属 | 是 | 高 | 中 | AI 需主动执行占领，不能只站在据点上 |
| 结算阶段 | 据点气运、状态倒计时、假身、狐火残留、勘验区、胜负 | 是 | 高 | 高 | 需固定处理顺序 |

结论：流程闭环成立。占领变为明确动作后，程序归属清晰，AI 评分表需要加入“执行占领”的收益判断。

# 13. AI 友好度总表

| 机制 | AI 判断难度 | 原因 | 建议评分逻辑 |
|---|---|---|---|
| AP 动作 | 中 | 动作选择多但 AP 少 | 枚举可行动作并评分 |
| 占点 | 中 | 需要主动占领，离开后归属保持 | 改变归属的占领动作高分，已经归属己方则低分 |
| 狐火 | 中 | 路径收益与攻击收益关联 | 经过残留/裂隙加分，充盈时优先攻击 |
| 狐印 | 中 | 目标价值提升 | 带狐印目标优先级提高，刷新不叠加 |
| 假身 | 中高 | 卡位与吸收价值抽象 | 优先放在 chokepoint、保护低血单位、支援目标旁 |
| 勘验区 | 中 | 区域覆盖选择 | 评分覆盖的狐火残留、裂隙、掩影、关键据点邻近格数量 |
| 律令 | 中 | 选择时机与三选一 | 根据敌方未来路线与当前局势评分 |
| 护阵 | 中 | 需要预测敌方攻击 | 保护低血、队长、占点单位；触发后清除 |
| 骰子 | 低 | 概率简单 | 使用期望成功数即可 |

总体评价：AI 可先实现规则启发式，不需要搜索树即可跑通 MVP。

# 15. 最终审计结论

当前 MVP 机制已经形成清晰闭环：

```text
流程闭环：
选择小队 → 部署 → 激活 → 行动 → 明确占领 → 掷骰 → 状态/资源 → 气运 → 胜负
```

```text
青丘闭环：
狐火残留 → 狐印 → 假身支援 → 狐火充盈 → 爆发
```

```text
天门闭环：
主动颁令 → 勘验区 → 护阵 → 阵地压制
```

程序友好度：高。

AI 友好度：中高。

当前规则量适合 MVP，不建议再增加新资源、新状态、新覆盖层或宝物系统。

下一步最应该做的是：

1. 数据结构字段表。
2. AI 行为评分表。
3. AI 对战测试与平衡指标。
