# 当前 MVP 范围

当前 MVP 目标：

- 验证规则是否成立。
- 验证两支小队是否有差异。
- 验证 AI 是否能正确操作。
- 验证 AI 自动对战能否用于平衡测试。

当前 MVP 模式：

- 自由对战。
- 玩家 vs AI。
- AI vs AI。

当前 MVP 内容：

- 2 支小队：青丘使团、天门执法队。
- 每名单位只保留 1 个核心技能。
- 小型六角格地图。
- 7 个基础地形。
- 覆盖层与区域：狐火残留、假身、勘验区。
- 状态：狐印、迷踪、逃逸。
- 资源：气运、狐火、AP。
- 律令系统。
- 战斗骰子系统。
- 占点系统。

当前 MVP 不包含：

- 单人战役。
- 教学关卡。
- 剧情推进。
- 解锁系统。
- 奖励系统。
- 成长系统。
- 经济系统。

# MVP 裁剪原则

1. 先保证自由对战能完整跑通。
2. 先验证 2 支小队是否有清晰差异。
3. 先让 AI 能完整跑完一局。
4. 先通过 AI vs AI 获取基础平衡数据。
5. 每名英雄只保留一个核心技能。
6. 地形、状态、资源、律令和骰子负责战术变化。
7. 百宝锦囊、宝物、妙计、阵型、法术、化形、天门威仪、命运系统暂不进入 MVP。
8. 单人战役、教学、解锁、奖励暂不进入 MVP。

# 文档读取规则

- `game_design/README.md` 是设定入口。
- 当前玩法规则以 `game_design/` 内文档为准。
- 程序实现以 `dev_design/` 内文档为准。
- 若开发文档与规则文档冲突，以 `game_design/规则总表.md` 为准。
- `archive/mvp_out_of_scope/` 只保存未来方向，不作为当前规则来源。
- 新设定进入 MVP 前，必须先更新 `设定审计总表.md` 与 `命名规范.md`。
