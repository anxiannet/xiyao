# 《夕妖》MVP 规则总表

## 文档目的

本文档是《夕妖》MVP 当前规则的统一审计入口。

用途：

- 汇总当前 MVP 的核心规则。
- 检查地形、状态、技能、宝物是否闭环。
- 作为开发实现和后续扩展战帮时的优先读取文档。

---

# 1. MVP 范围总览

当前 MVP 包含：

- 2 支战帮：天门执法队、青丘使团。
- 6 套百宝囊。
- 7 个基础地形标签。
- 3 种核心覆盖层。
- 核心标签系统。
- 队长唯一核心标记机制。
- 战斗骰子系统。
- 状态分类与友方净化规则。
- 3 个大回合。
- 气运胜利分。
- AI 对战。
- 教学战役与自由对战。

当前 MVP 不包含：

- 完整四象天气系统。
- 圣坛。
- 昼曜令台。
- 窄路。
- 临时障碍。
- 临时黄昏裂隙。
- 关闭裂隙。
- 直接改变基础地形的技能或宝物。
- 封界石。
- 联网天梯。
- 抽卡。
- 长期战帮养成。

# 3. 四象与命名总表

四象本源：

```text
昼曜
夜魄
晨辉
夕灵
```

对应关系：

```text
昼 ↔ 夜
晨 ↔ 夕
曜 ↔ 魄
辉 ↔ 灵
```

| 名词 | 类型 | 当前定义 | 是否 MVP 规则资源 |
|---|---|---|---|
| 夕灵 | 四象本源 | 黄昏、变化、灵性、化形与过渡的本源 | 否 |
| 狐火 | 青丘战帮资源 | 青丘通过收集狐火残留获得的资源 | 是 |
| 狐火残留 | 地格覆盖层 | 地面上的狐火痕迹，青丘经过时收集狐火 | 是 |
| 狐印 | 负面标记状态 | 青丘核心标记，青丘攻击带狐印目标时支援自动成功 | 是 |
| 威仪 | 天门战帮资源 | 天门通过律令、封印、净化、中央据点获得的资源 | 是 |
| 追缉 | 负面标记状态 | 天门核心标记，被天门攻击时 DEF_DICE -1 | 是 |
| 律令领域 | 地格覆盖层 | 天门法度形成的阵地覆盖层 | 是 |

不得再使用：

- 暮炁。
- 夕火作为资源名。
- 夕灵值作为 MVP 青丘资源。
- 狐火状态。

# 4. 队长唯一核心机制

## 4.1 青丘队长：苏绫

苏绫是青丘狐印的唯一稳定来源。

- 苏绫通过狐步、九尾掠影在特定地形出发并命中后施加狐印。
- 阿照、青萝、琉尾和百宝囊主要利用狐印，不稳定生产狐印。
- 苏绫阵亡后，青丘仍可使用狐火、假身、边缘机动，但狐印围猎能力明显下降。

## 4.2 天门队长：玄照

玄照是天门追缉的唯一稳定来源。

- 玄照通过宣令追缉施加追缉。
- 天门百宝囊主要利用追缉，不稳定生产追缉。
- 玄照阵亡后，天门仍可使用律令领域、封印、防守，但抓核心能力明显下降。

# 5. 通用状态规则

## 5.1 正面状态

正面状态通常只能施加给友方单位，除非效果明确写明可以给敌方。

MVP 示例：护盾。

## 5.2 负面状态

负面状态通常只能施加给敌方单位，除非效果明确写明可以给友方。

MVP 示例：狐印、封印、迷踪、易伤、追缉。

## 5.3 友方净化

当前 MVP 的“净化”默认指友方净化。

友方净化只能选择友方单位，并移除该友方单位身上的 1 个负面状态。

因此白烬的晨辉净化不能净化敌方单位身上的封印。

这不是“封印不可被任何净化移除”，而是当前 MVP 没有能选择敌方单位并移除负面状态的能力。

后续战帮若拥有“敌方净化”“驱散”“解除控制”等能力，并明确允许选择敌方单位，则可以移除封印。

## 5.4 默认持续时间

所有未注明持续时间的单位状态，默认持续 1 回合。

若文档特别写明“下一次移动前”“下一次受到攻击前”等触发式持续时间，则按具体文本执行。

# 9. 战斗骰子总表

| 骰面 | 数量 | 攻击时 | 防御时 |
|---|---:|---|---|
| 暴击 | 1 | 成功 | 成功 |
| 攻击 | 2 | 成功 | 失败 |
| 防御 | 2 | 失败 | 成功 |
| 支援 | 1 | 满足攻击支援则成功 | 满足防御支援则成功 |

攻击支援：被攻击目标周围 1 格内存在至少 1 个攻击方单位，且该单位不是本次攻击者。

防御支援：防御者周围 1 格内存在至少 1 个防御方友军。

特殊自动支援：

- 青丘攻击带狐印目标时，支援面自动成功。
- 天门单位位于律令领域内防御时，支援面自动成功。
- 假身可以参与攻击支援和防御支援。

# 15. 职责分离审计

## 已解决

1. 白烬不再清除覆盖层或生成律令领域。
2. 天律囊负责覆盖层战争。
3. 苏绫是狐印唯一稳定来源。
4. 玄照是追缉唯一稳定来源。
5. 青丘百宝囊不直接赠送狐火，资源回到地图收集。
6. 天门百宝囊不稳定生产追缉，只利用标记/追缉。
7. 通用规则开始使用标签：标记、召唤物、有害覆盖层、负面状态。

## 当前结论

MVP 可开发性：高。

战帮差异：明确。

后续扩展性：已从“具体名词耦合”升级为“标签驱动”。

# 16. 后续新增战帮规范

新增战帮时，必须先定义：

1. 队长专属机制。
2. 战帮资源。
3. 至少 1 个核心状态或覆盖层。
4. 上述对象的标签。
5. 与现有通用规则的交互。

新增技能或宝物时，应优先引用：

- 负面状态。
- 标记状态。
- 控制状态。
- 有害覆盖层。
- 召唤物。
- 战帮资源。

尽量避免在通用技能中引用具体名词：

- 狐印。
- 追缉。
