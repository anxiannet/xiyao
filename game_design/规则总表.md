# 1. MVP 范围总览

## 当前 MVP 包含

- 2 支战帮：天门执法队、青丘使团。
- 6 套百宝囊。
- 程序生成六角网格。
- 地形标签与覆盖层。
- 战斗骰子系统。
- 3 个大回合。
- 气运胜利分。
- AI 对战。
- 教学战役与自由对战。

## 当前 MVP 不包含

- 完整四象天气系统。
- 圣坛。
- 昼曜令台。
- 窄路。
- 障碍。
- 临时障碍。
- 复杂天气地形。
- 联网天梯。
- 抽卡。
- 长期战帮养成。
- 大量势力扩展。

## 4.4 假身归属建议规则

为避免“宝物招的假身是谁的假身”这类问题，建议立即采用以下统一规则：

| 项目 | 建议规则 |
|---|---|
| 阿照技能生成 | 阿照的假身 |
| 狐影铃生成 | 使用该宝物的青丘单位的假身 |
| 假身外观 | 复制生成者外观 |
| 假身名称 | “生成者名 + 假身”，例如“苏绫假身”“青萝假身” |
| 假身阵营 | 青丘 |
| 假身是否继承技能 | 不继承 |
| 假身是否继承属性 | 不继承，统一 HP 1 |
| 假身是否继承状态 | 不继承 |
| 假身是否提供支援 | 提供 |
| 假身是否可占点 | 不可占点 |
| 假身被封印 | 视为承受 1 次技能，封印后立即消失 |
| 假身被攻击 | 承受后立即消失 |

此规则尚需同步回青丘百宝囊与地形系统。

# 6. 战斗骰子总表

## 6.1 骰面

| 骰面 | 数量 | 攻击时 | 防御时 |
|---|---:|---|---|
| 暴击 | 1 | 成功 | 成功 |
| 攻击 | 2 | 成功 | 失败 |
| 防御 | 2 | 失败 | 成功 |
| 支援 | 1 | 满足攻击支援则成功 | 满足防御支援则成功 |

## 6.2 攻击支援

| 项目 | 规则 |
|---|---|
| 检查位置 | 被攻击目标周围 1 格 |
| 成功条件 | 至少 1 个攻击方单位在目标周围，且该单位不是本次攻击者 |
| 攻击者本人 | 不计入攻击支援 |
| 远程攻击者相邻 | 仍不计入攻击支援 |
| 假身 | 可以计入攻击支援 |
| 狐火自动支援 | 青丘攻击带狐火目标时，支援面自动成功 |

## 6.3 防御支援

| 项目 | 规则 |
|---|---|
| 检查位置 | 防御者周围 1 格 |
| 成功条件 | 至少 1 个防御方友军在防御者周围 |
| 防御者本人 | 不计入防御支援 |
| 假身 | 可以计入防御支援 |
| 律令领域自动支援 | 天门单位位于律令领域内防御时，支援面自动成功 |

## 6.4 结算

| 阶段 | 规则 |
|---|---|
| 暴击抵消 | 攻击暴击只能由防御暴击抵消 |
| 普通成功抵消 | 暴击抵消后比较普通成功数 |
| 命中条件 | 攻击方剩余成功数大于防御方剩余成功数 |
| 防住条件 | 攻击方剩余成功数小于或等于防御方剩余成功数 |
| 默认伤害 | 命中后造成基础伤害 |
| 额外成功伤害 | MVP 不默认采用“每多 1 成功 +1 伤害” |

# 13. 当前矛盾与待修正清单

## A级：会直接影响开发实现

### A1. 地形数量描述错误

地形文档写“MVP 只保留 5 种核心地形”，但实际列出：

```text
plain
central_objective
edge_objective
high_ground
cover_shadow
dusk_rift
```

实际是 6 个地形标签，其中 plain 是普通格。

建议修正为：

```text
MVP 保留 6 个基础地形标签，其中 plain 为普通格。
```

### A2. 裂隙针与地形范围冲突

青丘迷踪囊包含：

```text
裂隙针：打开一个临时黄昏裂隙 1 回合。
```

但地形文档将“临时裂隙”列为暂不进入 MVP。

二选一：

1. 删除裂隙针，换成已定义的位移/迷踪宝物。
2. 将“临时黄昏裂隙”纳入 MVP 覆盖层或临时地形。

建议：MVP 先删除裂隙针或改为“本回合将一个已有黄昏裂隙视为相邻”。

### A3. 狐影铃假身归属未明确

当前只写：

```text
在相邻格生成1个假身。
```

但未写是谁的假身。

建议立即补充：

```text
狐影铃生成使用者的假身。
假身复制使用者外观与名称，不复制技能、属性、状态。
统一 HP 1。
```

### A4. 灵狐羽引用未定义系统

灵狐羽写：

```text
从边缘格出发时额外不触发敌方反应。
```

但 MVP 没有“敌方反应”系统。

建议改为：

```text
从边缘格出发时，本次移动不触发追缉离开律令领域获得威仪的效果。
```

或直接删除该句。

### A5. 镜尾符的“增益外观”未定义

当前写：

```text
选择一个假身，使其复制相邻友方的增益外观，但不复制效果。
```

但没有“增益外观”UI/状态层定义。

建议改为：

```text
选择一个假身，使其显示为相邻友方单位的外观，直到被攻击、被技能指定或回合结束。
```

并明确这只是视觉伪装，不改变规则。

## B级：会影响平衡或审计，但不阻塞实现

### B1. 封印是否可净化未明确

白烬晨辉净化可净化：

- 狐火
- 迷踪
- 易伤
- 追缉

不包含封印。

需要明确：

- 封印不可净化，作为天门强控制。
- 或封印可净化，白烬也能解除。

建议：封印不可被晨辉净化移除，否则天门控制闭环过弱。

### B2. 狐火持续时间未统一

流程文档写狐火通常持续 1 回合，但各来源未全部写持续时间。

建议统一：

```text
狐火默认持续 1 回合。
来源未特别说明时均按 1 回合处理。
```

### B3. 护盾叠加规则未明确

白烬圣辉环护提供 1 护盾。

未说明是否可叠加。

建议：MVP 中护盾不叠加，已有护盾时刷新持续时间但不增加层数。

### B4. 律令领域持续时间不完全一致

默认持续 1 回合。

律令旗写“持续到下回合开始”。

建议统一术语：

```text
持续到施放方下次大回合开始。
```

或所有律令领域统一使用“剩余回合数”。

### B5. 琉尾资源倾向为夜魄

MVP 已归档完整四象天气系统，但琉尾仍写资源倾向：夜魄。

建议删除“资源倾向：夜魄”，或改为“无资源倾向”。

## C级：文案/命名层面

# 16. 结论

当前 MVP 的核心已经基本成型：

```text
青丘：狐火残留 → 夕灵 → 假身/支援/机动 → 边缘偷点或集火
天门：律令领域 → 威仪 → 追缉/封印/净化 → 中央压制和防御支援
```

主要问题不是大方向，而是少量旧文案、未定义触发条件和临时地形残留。

只要优先修正 A 级问题，当前规则即可进入可开发状态。
