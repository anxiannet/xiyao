# 《夕妖》MVP对局事件流

版本：v0.1.0-mvp  
更新时间：2026-06-18

本文档只定义后端运行时事件、日志与回放数据结构，不新增游戏规则。

---

## 1. 事件原则

每次动作必须生成事件。

事件用于：

- 对局回放
- 调试规则结算
- AI 决策审计
- 自动对战统计
- 前端只读展示

禁止：

- 前端直接写事件。
- 前端通过事件修改对局结果。
- API 绕过规则引擎直接写结果。
- 使用未定义事件类型。

---

## 2. 允许事件类型

当前只允许：

```text
match_start
round_start
turn_start
move
attack
skill
damage
status_add
status_remove
capture
turn_end
round_end
match_end
ai_decision
```

不得新增事件类型。

---

## 3. 事件表最小结构

对应运行表：

```text
xiyao_match_events
```

字段至少包含：

```ts
type MatchEvent = {
  id: string;
  match_id: string;
  round: number;
  turn: number;
  sequence: number;
  event_type:
    | 'match_start'
    | 'round_start'
    | 'turn_start'
    | 'move'
    | 'attack'
    | 'skill'
    | 'damage'
    | 'status_add'
    | 'status_remove'
    | 'capture'
    | 'turn_end'
    | 'round_end'
    | 'match_end'
    | 'ai_decision';
  source_unit_id: string | null;
  target_unit_id: string | null;
  source_tile_id: string | null;
  target_tile_id: string | null;
  payload: Record<string, unknown> | null;
  created_at: string;
};
```

`sequence` 在同一 `match_id` 内必须递增且唯一。

---

## 4. payload 使用边界

`payload` 只保存事件结果、调试信息和回放所需快照。

允许保存：

- 伤害值
- 命中结果
- 骰子结果
- AP 消耗
- 状态 ID
- 据点归属变化
- AI 候选动作评分
- 引擎版本
- 随机种子

禁止保存：

- 由前端传入的伤害结论
- 由前端传入的胜负结论
- 未经规则引擎校验的状态变化
- 新规则逻辑
- 新玩法字段

---

## 5. 对局创建事件流

创建对局时必须依次写入：

```text
1. match_start
2. round_start
3. turn_start
```

### 5.1 match_start

用途：记录对局创建。

建议 payload：

```ts
{
  mode: 'player_vs_ai' | 'ai_vs_ai',
  map_id: string,
  first_side: 'qingqiu' | 'tianmen',
  qingqiu_squad_id: string,
  tianmen_squad_id: string,
  engine_version: string,
  random_seed: string | null
}
```

### 5.2 round_start

用途：记录大回合开始。

建议 payload：

```ts
{
  round: number
}
```

### 5.3 turn_start

用途：记录行动方开始。

建议 payload：

```ts
{
  side: 'qingqiu' | 'tianmen',
  active_unit_ids: string[]
}
```

---

## 6. move 事件流

合法移动后至少写入：

```text
move
```

若移动触发地块或状态效果，继续写入：

```text
status_add
status_remove
```

示例顺序：

```text
move
status_add
status_remove
```

`move.payload` 建议：

```ts
{
  path_tile_ids: string[],
  mv_before: number,
  mv_after_modifier: number,
  ap_cost: number
}
```

---

## 7. attack 事件流

合法攻击后至少写入：

```text
attack
damage
```

可能继续写入：

```text
status_add
status_remove
match_end
```

示例顺序：

```text
attack
damage
status_add
status_remove
match_end
```

`attack.payload` 建议：

```ts
{
  atk_dice: number,
  def_dice: number,
  atk_success: number,
  def_success: number,
  hit: boolean,
  support_sources: string[],
  ap_cost: number
}
```

`damage.payload` 建议：

```ts
{
  damage: number,
  hp_before: number,
  hp_after: number,
  defeated: boolean,
  fortune_delta: number
}
```

---

## 8. skill 事件流

合法技能后至少写入：

```text
skill
```

根据技能结算结果，可能继续写入：

```text
damage
status_add
status_remove
capture
match_end
```

`skill.payload` 建议：

```ts
{
  skill_id: string,
  ap_cost: number,
  resolved_effect_keys: string[]
}
```

技能具体效果只能由规则引擎依据当前规则与配置执行。

---

## 9. capture 事件流

占领动作后写入：

```text
capture
```

无论成功或失败，都应记录事件，便于回放与 AI 分析。

`capture.payload` 建议：

```ts
{
  objective_tile_id: string,
  success: boolean,
  previous_owner: 'qingqiu' | 'tianmen' | null,
  next_owner: 'qingqiu' | 'tianmen' | null,
  actor_formal_unit_count: number,
  enemy_formal_unit_count: number,
  ap_cost: number
}
```

---

## 10. end_turn 事件流

结束行动后至少写入：

```text
turn_end
```

若进入新行动方：

```text
turn_start
```

若大回合结束：

```text
round_end
round_start
turn_start
```

若对局结束：

```text
match_end
```

示例：普通换边

```text
turn_end
turn_start
```

示例：大回合结束并进入下一回合

```text
turn_end
round_end
round_start
turn_start
```

示例：第三大回合结束并结算胜负

```text
turn_end
round_end
match_end
```

## 11. 状态事件

### 11.1 status_add

建议 payload：

```ts
{
  status_id: string,
  status_type: 'unit' | 'tile' | 'squad' | 'global',
  applied_to_unit_id: string | null,
  applied_to_tile_id: string | null,
  applied_to_side: 'qingqiu' | 'tianmen' | null,
  duration_scope: string | null
}
```

### 11.2 status_remove

建议 payload：

```ts
{
  status_id: string,
  status_type: 'unit' | 'tile' | 'squad' | 'global',
  removed_from_unit_id: string | null,
  removed_from_tile_id: string | null,
  removed_from_side: 'qingqiu' | 'tianmen' | null,
  remove_reason: string
}
```

## 12. AI决策事件

每次 AI 动作必须先写入：

```text
ai_decision
```

再通过规则引擎执行最终选择动作。

`ai_decision.payload` 必须包含：

```ts
{
  side: 'qingqiu' | 'tianmen',
  unit_id: string | null,
  candidates: Array<{
    action_type: 'move' | 'attack' | 'skill' | 'capture' | 'end_turn',
    unit_id: string | null,
    source_tile_id: string | null,
    target_tile_id: string | null,
    target_unit_id: string | null,
    skill_id: string | null,
    total_score: number,
    score_breakdown: Array<{
      key: string,
      value: number,
      reason: string
    }>
  }>,
  selected_index: number,
  scorer_version: string
}
```

评分来源必须是 `游戏/程序代码/docs/AI行为评分表.md`。

若评分表未建立，只能记录测试桩结果，不得作为稳定 AI 行为依据。

---

## 13. match_end 事件

对局结束时写入：

```text
match_end
```

建议 payload：

```ts
{
  winner_side: 'qingqiu' | 'tianmen' | 'draw' | null,
  end_reason: 'round_limit' | 'all_units_defeated' | 'auto_run_step_limit' | 'rule_error',
  qingqiu_fortune: number,
  tianmen_fortune: number,
  tie_breaker: string | null,
  round_count: number,
  turn_count: number
}
```

---

## 14. 回放读取

`GET /matches/:id/events` 返回按 `sequence` 升序排列的事件。

前端回放只能根据事件重建展示状态，不得以回放状态覆盖后端对局状态。

---

## 15. 统计读取

AI vs AI 统计来自两类数据：

1. `xiyao_match_events`：完整事件明细。
2. `xiyao_match_ai_stats`：每局汇总结果。

统计字段必须至少包含：

```text
胜方
回合数
击杀数
占点数
技能次数
状态触发次数
律令次数
地图ID
先手方
```
