# 《夕妖》MVP开发文档

本目录用于存放面向程序实现、数据库、AI、自动对战、日志、测试与工程落地的文档。

与 `游戏/游戏设定/` 的区别：

- `游戏/游戏设定/`：设定文档，回答“游戏是什么、规则是什么、角色是谁”。
- `游戏/程序代码/docs/`：开发文档，回答“程序如何存储、如何运行、如何测试、如何调试”。

# 文档边界

## 应放入 `游戏/程序代码/docs/`

- 数据结构
- 数据库表设计
- API 设计
- 前端状态结构
- 规则引擎结构
- AI 行为评分表
- AI 自动对战测试
- 平衡测试指标
- 日志与回放格式
- 技术实现任务单

## 不应放入 `游戏/程序代码/docs/`

- 世界观正文
- 人物志
- 小队背景
- 技能文案设定
- 战役剧情
- 命名规范
- 规则总表正文

这些内容应放在 `游戏/游戏设定/`。

---

# 读取顺序

实现功能时，建议顺序：

1. 先读 `游戏/游戏设定/README.md`，确认当前 MVP 规则范围。
2. 再读 `游戏/游戏设定/规则总表.md`，确认规则本体。
3. 再读本目录对应开发文档，确认数据结构与实现方式。
