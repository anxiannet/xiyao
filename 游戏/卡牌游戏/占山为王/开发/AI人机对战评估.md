# 《夕妖：占山为王》AI人机对战评估

状态：开发评估（V0.1）

## 文档边界

本文只讨论 AI、人机对战与程序实现方向。

不维护正式玩法规则、不维护卡牌效果正文、不维护世界观设定。

正式玩法规则以：

```text
游戏/卡牌游戏/占山为王/README.md
```

为准。

---

## 结论

《夕妖：占山为王》当前规则对 AI 友好。

原因：

- 每回合只打 1 张牌
- 没有生命值
- 没有妖气资源
- 没有残血记录
- 没有棋盘路径
- 没有多段连锁
- 胜利条件明确：拥有 3 块地盘

AI 可以通过简单优先级规则完成可玩的人机对战。

```text
players[]
  hand[]
  lands[]
  treasures[]

central_lands[]
deck[]
discard[]
current_player_index
```

### 玩家状态

每名玩家至少包含：

- 手牌
- 已拥有地盘
- 已生效法宝

### 公共状态

公共状态至少包含：

- 中央地盘区
- 牌库
- 弃牌堆
- 当前行动玩家

## AI决策优先级草案

基础 AI 可按以下优先级行动：

1. 如果自己打出某张牌后能立刻拥有 3 块地盘，则优先打出该牌。
2. 如果任意对手已经拥有 2 块地盘，则优先阻止该对手。
3. 如果中央地盘区有可获得地盘，优先获得地盘。
4. 如果无法获得地盘，优先抢夺地盘最多玩家的地盘。
5. 如果没有合适地盘目标，优先偷牌或弃牌干扰。
6. 如果有防御法宝，优先保留而非随意打出。
7. 若无有效行动，打出最低价值牌或跳过。

---

## 推荐卡牌效果动作集合

为了降低 AI 与程序实现复杂度，卡牌效果应优先限制在以下动作内：

| 动作 | 含义 |
|---|---|
| gain_land | 从中央地盘区获得 1 块地盘 |
| steal_land | 抢夺其他玩家 1 块地盘 |
| discard_card | 令目标玩家弃牌 |
| steal_card | 偷取目标玩家手牌 |
| cancel_effect | 取消一次效果 |
| reflect_effect | 反弹一次效果 |
| protect_land | 保护一块地盘 |
| swap_land | 交换两块地盘 |

暂不建议加入：

- 复杂连锁
- 长期状态叠加
- 玩家谈判
- 隐藏身份
- 大量随机目标

## AI评分维度草案

AI 对每个可行动作计算分数。

建议基础分：

| 行为 | 基础分 |
|---|---:|
| 自己获得第 3 块地盘 | 1000 |
| 阻止对手获得第 3 块地盘 | 900 |
| 自己获得第 2 块地盘 | 300 |
| 抢夺当前领先玩家地盘 | 250 |
| 获得第 1 块地盘 | 180 |
| 偷取领先玩家手牌 | 120 |
| 让领先玩家弃牌 | 100 |
| 打出防御法宝 | 80 |
| 无目标低价值行动 | 10 |

## 人机对战实现建议

第一阶段只做规则型 AI。

不需要机器学习。

不需要大模型推理。

不需要复杂搜索。

推荐实现方式：

```text
枚举当前手牌可用行动
↓
为每个行动生成候选目标
↓
按评分表计算分数
↓
选择最高分行动
↓
执行
```
