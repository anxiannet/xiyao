# 《夕妖》MVP运行表结构

版本：v0.1.0-mvp  
更新时间：2026-06-18

本文档是 MVP 自由对战运行表结构草案。运行表只保存对局运行状态、事件日志、AI统计与回放数据，不保存游戏规则正文，不写入新玩法。

---

## 1. 命名规范

运行表统一使用：

```text
xiyao_match_*
```

当前必须建立：

```text
xiyao_match_matches
xiyao_match_units
xiyao_match_tiles
xiyao_match_statuses
xiyao_match_events
xiyao_match_ai_stats
```

---

## 2. 通用字段原则

所有运行表建议使用：

```sql
id uuid primary key default gen_random_uuid()
created_at timestamptz not null default now()
updated_at timestamptz not null default now()
```

若数据库环境不支持 `gen_random_uuid()`，迁移脚本中再按实际 Supabase 扩展处理。

禁止：

- 使用万能 `json` 替代结构化字段。
- 在字段中写规则逻辑。
- 绑定具体小队专属规则。
- 保存前端传入的结算结果。

例外：

- `xiyao_match_events.payload` 可用于事件调试与回放。
- `xiyao_match_ai_stats.summary_payload` 可用于批量统计补充信息。

这两个 JSON 字段不得承载规则判断。

## 3. xiyao_match_matches

用途：保存一局对局的主状态。

```sql
create table xiyao_match_matches (
  id uuid primary key default gen_random_uuid(),

  mode text not null,
  map_id text not null,

  qingqiu_squad_id text not null,
  tianmen_squad_id text not null,

  first_side text not null,
  current_side text not null,

  round_no integer not null default 1,
  turn_no integer not null default 1,

  status text not null default 'active',
  winner_side text null,
  end_reason text null,

  qingqiu_fortune integer not null default 0,
  tianmen_fortune integer not null default 0,
  qingqiu_foxfire integer not null default 0,

  random_seed text null,
  engine_version text not null,
  ai_version text null,

  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint xiyao_match_matches_mode_check
    check (mode in ('player_vs_ai', 'ai_vs_ai')),

  constraint xiyao_match_matches_side_check
    check (first_side in ('qingqiu', 'tianmen')),

  constraint xiyao_match_matches_current_side_check
    check (current_side in ('qingqiu', 'tianmen')),

  constraint xiyao_match_matches_status_check
    check (status in ('active', 'finished', 'error')),

  constraint xiyao_match_matches_winner_side_check
    check (winner_side is null or winner_side in ('qingqiu', 'tianmen', 'draw')),

  constraint xiyao_match_matches_round_check
    check (round_no >= 1 and round_no <= 3),

  constraint xiyao_match_matches_foxfire_check
    check (qingqiu_foxfire >= 0 and qingqiu_foxfire <= 5)
);
```

说明：

- `mode` 只允许 `player_vs_ai`、`ai_vs_ai`。
- `round_no` 当前 MVP 限制为 1 到 3。
- `qingqiu_foxfire` 是青丘共享资源运行值，不代表规则逻辑。
- 胜负结算仍由规则引擎执行。

建议索引：

```sql
create index idx_xiyao_match_matches_status on xiyao_match_matches(status);
create index idx_xiyao_match_matches_map_id on xiyao_match_matches(map_id);
create index idx_xiyao_match_matches_created_at on xiyao_match_matches(created_at);
```

## 4. xiyao_match_units

用途：保存对局内单位快照与运行状态。

```sql
create table xiyao_match_units (
  id uuid primary key default gen_random_uuid(),

  match_id uuid not null references xiyao_match_matches(id) on delete cascade,

  cfg_unit_id text not null,
  side text not null,
  unit_name text not null,

  unit_kind text not null default 'hero',
  is_leader boolean not null default false,
  is_formal_unit boolean not null default true,

  tile_id uuid null,

  hp integer not null,
  max_hp integer not null,
  ap_remaining integer not null default 2,

  atk integer not null,
  def integer not null,
  mv integer not null,
  rng integer not null,
  dmg integer not null,

  alive boolean not null default true,
  acted boolean not null default false,

  summoned_by_unit_id uuid null references xiyao_match_units(id),
  summon_duration_rounds integer null,
  absorbed_hit_count integer not null default 0,

  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint xiyao_match_units_side_check
    check (side in ('qingqiu', 'tianmen')),

  constraint xiyao_match_units_kind_check
    check (unit_kind in ('hero', 'summon')),

  constraint xiyao_match_units_hp_check
    check (hp >= 0 and max_hp >= 1),

  constraint xiyao_match_units_ap_check
    check (ap_remaining >= 0)
);
```

说明：

- `cfg_unit_id` 指向配置单位 ID。
- `tile_id` 指向本局 `xiyao_match_tiles.id`。
- 假身等召唤物使用 `unit_kind = 'summon'`。
- 占领统计依赖 `is_formal_unit`，规则仍由引擎判断。

建议索引：

```sql
create index idx_xiyao_match_units_match_id on xiyao_match_units(match_id);
create index idx_xiyao_match_units_side on xiyao_match_units(match_id, side);
create index idx_xiyao_match_units_tile_id on xiyao_match_units(tile_id);
```

## 5. xiyao_match_tiles

用途：保存对局地图格子快照与据点归属。

```sql
create table xiyao_match_tiles (
  id uuid primary key default gen_random_uuid(),

  match_id uuid not null references xiyao_match_matches(id) on delete cascade,

  cfg_tile_id text not null,
  map_id text not null,

  q integer not null,
  r integer not null,

  terrain_id text not null,

  deploy_side text null,
  objective_kind text null,
  objective_owner text null,

  blocks_movement boolean not null default false,
  blocks_line_of_sight boolean not null default false,

  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint xiyao_match_tiles_deploy_side_check
    check (deploy_side is null or deploy_side in ('qingqiu', 'tianmen')),

  constraint xiyao_match_tiles_objective_kind_check
    check (objective_kind is null or objective_kind in ('central', 'edge')),

  constraint xiyao_match_tiles_objective_owner_check
    check (objective_owner is null or objective_owner in ('qingqiu', 'tianmen')),

  constraint xiyao_match_tiles_unique_coord
    unique (match_id, q, r),

  constraint xiyao_match_tiles_unique_cfg_tile
    unique (match_id, cfg_tile_id)
);
```

说明：

- 地图格子在创建对局时从配置表复制为快照。
- `objective_owner` 只保存当前归属，不保存占领规则。
- `blocks_movement`、`blocks_line_of_sight` 是地形运行属性快照，具体判断仍由引擎执行。

建议索引：

```sql
create index idx_xiyao_match_tiles_match_id on xiyao_match_tiles(match_id);
create index idx_xiyao_match_tiles_objective on xiyao_match_tiles(match_id, objective_kind);
```

## 6. xiyao_match_statuses

用途：保存单位、地块、小队、全局状态实例。

```sql
create table xiyao_match_statuses (
  id uuid primary key default gen_random_uuid(),

  match_id uuid not null references xiyao_match_matches(id) on delete cascade,

  cfg_status_id text not null,
  status_type text not null,

  unit_id uuid null references xiyao_match_units(id) on delete cascade,
  tile_id uuid null references xiyao_match_tiles(id) on delete cascade,
  side text null,

  source_unit_id uuid null references xiyao_match_units(id),

  round_applied integer not null,
  turn_applied integer not null,
  expires_round integer null,
  expires_turn integer null,

  stack_count integer not null default 1,
  active boolean not null default true,

  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint xiyao_match_statuses_type_check
    check (status_type in ('unit', 'tile', 'squad', 'global')),

  constraint xiyao_match_statuses_side_check
    check (side is null or side in ('qingqiu', 'tianmen')),

  constraint xiyao_match_statuses_stack_check
    check (stack_count >= 1)
);
```

说明：

- 单位状态使用 `unit_id`。
- 地块状态使用 `tile_id`。
- 小队状态使用 `side`。
- 全局状态不绑定具体单位或地块。
- 状态持续、刷新、移除由规则引擎执行。

建议索引：

```sql
create index idx_xiyao_match_statuses_match_id on xiyao_match_statuses(match_id);
create index idx_xiyao_match_statuses_unit on xiyao_match_statuses(unit_id);
create index idx_xiyao_match_statuses_tile on xiyao_match_statuses(tile_id);
create index idx_xiyao_match_statuses_active on xiyao_match_statuses(match_id, active);
```

## 7. xiyao_match_events

用途：保存事件日志与回放数据。

```sql
create table xiyao_match_events (
  id uuid primary key default gen_random_uuid(),

  match_id uuid not null references xiyao_match_matches(id) on delete cascade,

  round_no integer not null,
  turn_no integer not null,
  sequence integer not null,

  event_type text not null,

  source_unit_id uuid null references xiyao_match_units(id),
  target_unit_id uuid null references xiyao_match_units(id),
  source_tile_id uuid null references xiyao_match_tiles(id),
  target_tile_id uuid null references xiyao_match_tiles(id),

  payload jsonb null,

  created_at timestamptz not null default now(),

  constraint xiyao_match_events_unique_sequence
    unique (match_id, sequence),

  constraint xiyao_match_events_type_check
    check (event_type in (
      'match_start',
      'round_start',
      'turn_start',
      'move',
      'attack',
      'skill',
      'damage',
      'status_add',
      'status_remove',
      'capture',
      'turn_end',
      'round_end',
      'match_end',
      'ai_decision'
    ))
);
```

说明：

- `payload` 只用于事件结果、调试与回放。
- `payload` 不得承载规则逻辑。
- 所有动作必须至少生成一个事件。

建议索引：

```sql
create index idx_xiyao_match_events_match_sequence on xiyao_match_events(match_id, sequence);
create index idx_xiyao_match_events_type on xiyao_match_events(match_id, event_type);
create index idx_xiyao_match_events_created_at on xiyao_match_events(created_at);
```

## 8. xiyao_match_ai_stats

用途：保存 AI vs AI 单局统计与批量统计基础数据。

```sql
create table xiyao_match_ai_stats (
  id uuid primary key default gen_random_uuid(),

  match_id uuid not null references xiyao_match_matches(id) on delete cascade,

  batch_id uuid null,

  map_id text not null,
  first_side text not null,
  winner_side text null,

  round_count integer not null,
  turn_count integer not null,
  step_count integer not null,

  qingqiu_fortune integer not null,
  tianmen_fortune integer not null,

  kill_count integer not null default 0,
  capture_count integer not null default 0,
  skill_count integer not null default 0,
  status_trigger_count integer not null default 0,
  decree_count integer not null default 0,

  qingqiu_kill_count integer not null default 0,
  tianmen_kill_count integer not null default 0,
  qingqiu_capture_count integer not null default 0,
  tianmen_capture_count integer not null default 0,
  qingqiu_skill_count integer not null default 0,
  tianmen_skill_count integer not null default 0,

  random_seed text null,
  engine_version text not null,
  ai_version text not null,

  summary_payload jsonb null,

  created_at timestamptz not null default now(),

  constraint xiyao_match_ai_stats_first_side_check
    check (first_side in ('qingqiu', 'tianmen')),

  constraint xiyao_match_ai_stats_winner_side_check
    check (winner_side is null or winner_side in ('qingqiu', 'tianmen', 'draw')),

  constraint xiyao_match_ai_stats_count_check
    check (
      round_count >= 0 and
      turn_count >= 0 and
      step_count >= 0 and
      kill_count >= 0 and
      capture_count >= 0 and
      skill_count >= 0 and
      status_trigger_count >= 0 and
      decree_count >= 0
    )
);
```

说明：

- 每一局 AI vs AI 结束后写入一条。
- 1000 局测试按 `batch_id` 聚合。
- `summary_payload` 只放补充统计，不放规则逻辑。

建议索引：

```sql
create index idx_xiyao_match_ai_stats_batch_id on xiyao_match_ai_stats(batch_id);
create index idx_xiyao_match_ai_stats_map_id on xiyao_match_ai_stats(map_id);
create index idx_xiyao_match_ai_stats_winner on xiyao_match_ai_stats(winner_side);
create index idx_xiyao_match_ai_stats_created_at on xiyao_match_ai_stats(created_at);
```
